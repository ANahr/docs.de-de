# Erweitern der Windows Workflow Foundation
Im folgenden Abschnitt wird beschrieben, wie [!INCLUDE[wf](../../../includes/wf-md.md)] mit benutzerdefinierten Aktivitäten und Designern in neu gehosteten Umgebungen außerhalb von [!INCLUDE[vs2010](../../../includes/vs2010-md.md)] erweitert werden kann.  
  
## In diesem Abschnitt  
 [Anpassen des Workflowentwurfsvorgangs](../../../docs/framework/windows-workflow-foundation//customizing-the-workflow-design-experience.md)  
 Gibt an, wie die Szenarien für das Entwerfen benutzerdefinierter Aktivitäten und das erneute Hosten des [!INCLUDE[wfd1](../../../includes/wfd1-md.md)] in [!INCLUDE[netfx40_short](../../../includes/netfx40-short-md.md)] stark vereinfacht wurden.Entwicklung und Bereitstellung sind jetzt einfacher und flexibler, da das neue Programmiermodell mit dem Aktivitätsdesigner auf [!INCLUDE[avalon1](../../../includes/avalon1-md.md)] basiert.
