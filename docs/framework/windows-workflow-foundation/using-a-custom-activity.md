# <a name="using-a-custom-activity"></a><span data-ttu-id="c9796-102">Verwenden einer benutzerdefinierten Aktivität</span><span class="sxs-lookup"><span data-stu-id="c9796-102">Using a custom activity</span></span>
<span data-ttu-id="c9796-103">Aktivitäten, die von <xref:System.Activities.Activity> oder den Unterklassen abgeleitet werden, können zu größeren Workflows zusammengesetzt oder direkt im Code erstellt werden.</span><span class="sxs-lookup"><span data-stu-id="c9796-103">Activities that derive from <xref:System.Activities.Activity> or its subclasses can be composed into larger workflows, or created directly in code.</span></span> <span data-ttu-id="c9796-104">In diesem Thema wird beschrieben, wie Sie benutzerdefinierte Aktivitäten in Workflows verwenden, die entweder im Code oder im Designer erstellt wurden.</span><span class="sxs-lookup"><span data-stu-id="c9796-104">This topic describes how to use custom activities in workflows created either in code or in the designer.</span></span>  
  
> [!NOTE]
>  <span data-ttu-id="c9796-105">Benutzerdefinierte Aktivitäten können im selben Projekt, in dem sie definiert sind, verwendet werden, solange die benutzerdefinierte Aktivität und der Aktivität, die verwendet werden (d. h. von einem instanziierenden Typ vom Buildprozess generierten geladen) kompiliert werden, wenn die verweisende Aktivität geladen wird dynamisch (z. B. mit ActivityXAMLServices), klicken Sie dann die referenzierte Assembly in einem anderen Projekt platziert werden soll, oder der XAML-Designer generierter Code muss manuell bearbeitet werden, um dies zu ermöglichen.</span><span class="sxs-lookup"><span data-stu-id="c9796-105">Custom activities can be used in the same project in which they are defined, as long as both the custom activity and the activity that uses it are compiled (i.e. loaded by an instantiating type generated by the build process) If the referencing activity is loaded dynamically (e.g. using ActivityXAMLServices), then the referenced assembly should be placed in a different project, or the designer-generated XAML needs to be hand-edited to enable this.</span></span>  
  
#### <a name="using-a-custom-activity-to-a-workflow-project"></a><span data-ttu-id="c9796-106">Verwenden einer benutzerdefinierten Aktivität in einem Workflowprojekt</span><span class="sxs-lookup"><span data-stu-id="c9796-106">Using a custom activity to a workflow project</span></span>  
  
1.  <span data-ttu-id="c9796-107">Fügen Sie einen Verweis vom Hostprojekt zum Aktivitätsbibliotheksprojekt hinzu, in dem die benutzerdefinierte Aktivität enthalten ist.</span><span class="sxs-lookup"><span data-stu-id="c9796-107">Add a reference from the host project to the activity library project containing the custom activity.</span></span>  
  
2.  <span data-ttu-id="c9796-108">Erstellen Sie die Projektmappe.</span><span class="sxs-lookup"><span data-stu-id="c9796-108">Build the solution.</span></span>  
  
3.  <span data-ttu-id="c9796-109">Um die benutzerdefinierte Aktivität im Designer zu verwenden, identifizieren Sie die benutzerdefinierte Aktivität in der Toolbox, und ziehen Sie die Aktivität auf die Designeroberfläche.</span><span class="sxs-lookup"><span data-stu-id="c9796-109">To use the custom activity in the designer, locate the custom activity in the toolbox, and drag the activity onto the designer surface.</span></span>  
  
4.  <span data-ttu-id="c9796-110">Um die benutzerdefinierte Aktivität im Code zu verwenden, fügen Sie eine Using-Anweisung hinzu, die auf das benutzerdefinierte Aktivitätsprojekt verweist, und übergeben Sie eine neue Instanz der Aktivität an <xref:System.Activities.WorkflowInvoker.Invoke%2A>.</span><span class="sxs-lookup"><span data-stu-id="c9796-110">To use the custom activity in code, add a Using statement that refers to the custom activity project, and pass a new instance of the activity to <xref:System.Activities.WorkflowInvoker.Invoke%2A>.</span></span>
