# <a name="wpf-community-feedback"></a>WPF-Communityfeedback
[!INCLUDE[TLA#tla_ms](../../../../includes/tlasharptla-ms-md.md)] stellt eine Vielzahl von Communityressourcen für Sie bereit, mit denen Sie sich über [!INCLUDE[TLA#tla_wpf](../../../../includes/tlasharptla-wpf-md.md)] informieren, austauschen und Feedback senden können. Zu diesen Ressourcen gehören die von [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)]-Teammitgliedern und Communitysprechern veröffentlichten Blogs, Foren und das [!INCLUDE[TLA2#tla_ms](../../../../includes/tla2sharptla-ms-md.md)] Produktfeedbackcenter. Die einzelnen Communityressourcen bieten unterschiedliche Vorteile. Diese Vorteile werden hier erläutert. Außerdem werden bewährte Vorgehensweisen für die Verwendung der einzelnen Ressourcen beschrieben, die sicherstellen, dass Sie eine optimale Reaktion von der Community insgesamt und [!INCLUDE[TLA2#tla_ms](../../../../includes/tla2sharptla-ms-md.md)] im Besonderen erhalten.  
  
> [!NOTE]
>  Senden Sie uns Produktfeedback nicht über den Link **Kommentare** unten auf jeder Seite. Diese Links sind nur für das Feedback zur Dokumentation gedacht.  
  
 
  
<a name="Blogs"></a>   
## <a name="blogs"></a>Blogs  
 Mitglieder des [!INCLUDE[TLA2#tla_ms](../../../../includes/tla2sharptla-ms-md.md)] [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)]-Teams und Communityleiter erstellen eine Vielzahl von [Blogs](http://go.microsoft.com/fwlink/?LinkId=98852), die sich auf [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)] im Allgemeinen oder bestimmte Teilmengen von [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)] im Besonderen beziehen. Die Inhalte sind breit gefächert und erstrecken sich im Allgemeinen von Produktentwicklungen, Motivation und Philosophien bis hin zur detaillierten Erläuterung bestimmter Szenarien, Probleme und der [!INCLUDE[TLA#tla_api](../../../../includes/tlasharptla-api-md.md)]. Zusätzlich können Sie über Beiträge, die Sie interessieren, mit anderen, die sich mit der Plattform sehr gut auskennen, diskutieren oder weitere Fragen dazu stellen.  
  
<a name="Forums"></a>   
## <a name="forums"></a>Foren  
 Das [WPF-Forum](http://go.microsoft.com/fwlink/?LinkID=187440) (und das [Forum](http://go.microsoft.com/fwlink/?LinkId=98855) für [!INCLUDE[TLA#tla_winvista](../../../../includes/tlasharptla-winvista-md.md)] 64) ist die primäre Communityressource zur Diskussion und Behebung von Problemen. Foren vereinfachen die Diskussion und Problemlösung, indem sie ein umfassendes Angebot unterstützender Features bereitstellen, die Folgendes umfassen:  
  
-   Suchfunktionen  
  
-   Verfolgung von Diskussionen  
  
-   Umfangreiche Formatierung für Text und Code  
  
-   Integration in [!INCLUDE[TLA#tla_visualstu](../../../../includes/tlasharptla-visualstu-md.md)]  
  
-   Einbeziehung von MVPs (Most Valued Professionals) und der Community  
  
-   Überwachung, um sicherzustellen, dass auf Beiträge schnellstmöglich reagiert wird  
  
 Auch wenn sich eine Frage auf [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)] bezieht, ist das [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)]-Forum möglicherweise nicht immer der am besten geeignete Ort, um darüber zu diskutieren. Ihre Frage könnte sich beispielsweise auf ein spezielles [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)]-Tool wie [Expression Blend](http://go.microsoft.com/fwlink/?LinkId=98858), [Expression Web](http://go.microsoft.com/fwlink/?LinkId=98859) und [Expression Design beziehen](http://go.microsoft.com/fwlink/?LinkId=98860). In diesen Fällen sollten Sie die entsprechenden [Expression-Communityforen](http://go.microsoft.com/fwlink/?LinkId=98861) nutzen.  
  
### <a name="forum-best-practices"></a>Bewährte Vorgehensweisen in Foren  
 Durch die Befolgen der folgenden bewährten Vorgehensweisen können Sie dazu beitragen, dass im [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)]-Forum veröffentlichte Fragen schnellstmöglich beantwortet werden. Diese Vorgehensweisen gelten für alle Foren.  
  
#### <a name="search-existing-posts"></a>Durchsuchen vorhandener Beiträge  
 Einige Probleme treten häufiger auf, sodass andere Benutzer ihnen möglicherweise bereits begegnet sind. Folglich können Sie das Problem schnell lösen oder Ihren Beitrag einer vorhandenen Diskussion hinzufügen.  
  
#### <a name="use-meaningful-titles"></a>Verwenden aussagekräftiger Titel  
 Präzise, aussagekräftige Titel verbessern die Auffindbarkeit der Beiträge. Außerdem können die anderen Communitymitglieder im [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)]-Forum so einfacher feststellen, ob sie Ihr Problem lösen können.  
  
#### <a name="include-appropriate-content"></a>Einschließen geeigneter Inhalte  
 Sie sollten das Problem beschreiben und erläutern, wie Sie versucht haben, es zu beheben. Falls möglich, sollten Sie unterstützende Codeausschnitte oder ein möglichst einfaches Beispiel hinzufügen, das Ihr Problem veranschaulicht. Diese Einzelheiten erhöhen die Chance, dass Ihre Frage rasch beantwortet wird.  
  
 Eine umfassende Auflistung bewährter Vorgehensweisen für Foren finden Sie [online](http://www.microsoft.com/wn3/locales/help/help_en-us.htm).  
  
<a name="Product_Feedback_Center"></a>   
## <a name="product-feedback-center"></a>Produktfeedbackcenter  
 Mitunter lassen sich Probleme nur schwierig oder gar nicht beheben. Ursache solcher Situationen können Probleme in der Technologie, Schwierigkeiten der Anwendung der Technologie auf bestimmte Szenarien oder eine mangelnde Unterstützung bestimmter Szenarien sein. Diese Informationen sind wichtig für [!INCLUDE[TLA2#tla_ms](../../../../includes/tla2sharptla-ms-md.md)] und können über das [Produktfeedbackcenter](http://go.microsoft.com/fwlink/?LinkId=99381) mitgeteilt werden.  
  
 An das [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)]-Produktfeedbackcenter gesendete Informationen werden an die interne Fehlerdatenbank des [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)]-Teams weitergeleitet. Dies ist die zuverlässigste Möglichkeit, dem Zuständigen für das jeweilige [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)]-Feature Ihr Feedback zu geben. Darüber hinaus können Sie Empfehlungen und Fehler überprüfen, nachverfolgen und mit Bewertungen versehen, damit das [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)]-Team Probleme priorisieren kann.  
  
### <a name="wpf-product-feedback-center-best-practices"></a>Bewährte Methoden für das WPF-Produktfeedbackcenter  
 Beim Veröffentlichen von Beiträgen im [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)]-Produktfeedbackcenter empfiehlt sich unbedingt das Durchsuchen vorhandener Beiträge, die Angabe eines aussagekräftigen Titels und das Einfügen geeigneter Inhalte. Dies entspricht dem Veröffentlichen von Beiträgen im [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)]-Forum. Die folgenden bewährten Vorgehensweisen werden zusätzlich empfohlen.  
  
#### <a name="search-existing-posts"></a>Durchsuchen vorhandener Beiträge  
 Einige Probleme treten häufiger auf, sodass andere Benutzer ihnen möglicherweise bereits begegnet sind. Folglich können Sie das Problem schnell lösen oder Ihren Beitrag einer vorhandenen Diskussion hinzufügen.  
  
#### <a name="use-meaningful-titles"></a>Verwenden aussagekräftiger Titel  
 Präzise, aussagekräftige Titel verbessern die Chance, dass Ihr Problem in kürzestmöglicher Zeit an das am besten geeignete [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)]-Team weitergeleitet wird. Dies ist für eine Technologie wie [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)], die viele zusammenhängende Features enthält, besonders wichtig.  
  
#### <a name="describe-how-to-reproduce-your-bug"></a>Beschreiben der Reproduktion Ihres Fehlers  
 Wenn Sie einen Fehler melden, müssen Sie die folgenden Angaben machen, sofern relevant:  
  
-   Geben Sie eine klare Beschreibung des Fehlers.  
  
-   Fügen Sie Codeausschnitte hinzu, um die Fehlerbeschreibung zu ergänzen.  
  
-   Listen Sie die Schritte auf, die veranschaulichen, wie sich der Fehler reproduzieren lässt.  
  
-   Schließen Sie ein möglichst kleines Codebeispiel ein, mit dem der Fehler reproduziert wird.  
  
-   Geben Sie an, ob sich der Fehler durchgängig reproduzieren lässt oder nicht.  
  
-   Fügen Sie relevante Ausnahmeinformationen hinzu.  
  
 Wenn der Fehler im Zusammenhang mit der Installation oder dem Setup steht, fügen Sie die relevanten Installationsprotokolle und Momentaufnahmen an (Dateien mit dem Präfix „dd_“, die sich im Ordner %temp% befinden).  
  
 Bei Kompilierungs- oder Buildproblemen fügen Sie die Buildprotokolle an. Das MSBuild-System kann so konfiguriert werden, dass die Protokollierung mit verschiedenen Ausführlichkeitsgraden unterstützt wird. Dazu wird die Option „/v:“ in der Befehlszeile verwendet oder der gewünschte Ausführlichkeitsgrad über eine [!INCLUDE[TLA#tla_ide](../../../../includes/tlasharptla-ide-md.md)] wie [!INCLUDE[TLA#tla_visualstu](../../../../includes/tlasharptla-visualstu-md.md)] konfiguriert.  
  
#### <a name="provide-environment-information"></a>Bereitstellen von Umgebungsinformationen  
 Hintergrundinformationen können oft nützlich sein, um Ihren Beitrag mit Kontext zu versehen. Insbesondere ist es wichtig, die Betriebssystemplattform, Prozessorfamilie und -architektur anzugeben, z. B. "Windows XP SP2, Pentium III, x86".  
  
 Wenn sich das Problem, das Sie melden, auf das Rendering bezieht, sollten Sie nach Möglichkeit auch Angaben zu Grafikkarte und Treiber machen. Diese Informationen sind wichtig, da [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)] ein Präsentationsframework ist.  
  
#### <a name="provide-solution-or-project-information"></a>Angeben von Lösungs- oder Projektinformationen  
 Fehler können auf zur Entwicklung und Erstellung von Anwendungen verwendete Tools und die zu erstellenden Anwendungstypen bezogen sein. Daher können folgende Angaben nützlich sein:  
  
-   Der Typ der Anwendung, den Sie erstellen, z. B.:  
  
    -   Anwendung (.exe) oder Bibliothek (.dll)  
  
    -   [!INCLUDE[TLA#tla_xaml](../../../../includes/tlasharptla-xaml-md.md)]-Browseranwendung (XBAP)  
  
    -   Lose [!INCLUDE[TLA2#tla_xaml](../../../../includes/tla2sharptla-xaml-md.md)] Anwendung  
  
    -   Eigenständige installierte Anwendungen  
  
    -   Eigenständige mit ClickOnce bereitgestellte Anwendungen  
  
-   Das Entwicklungstool, z. B.:  
  
    -   MSBuild  
  
    -   Expression Graphic Designer  
  
    -   Expression Interactive Designer  
  
    -   [!INCLUDE[TLA#tla_visualstu](../../../../includes/tlasharptla-visualstu-md.md)]  
  
-   Die Projektmappenkonfiguration, z. B.:  
  
    -   Projektmappe  
  
    -   Einzelnes Projekt  
  
    -   Eine Projektmappe mit mehreren abhängigen Projekten  
  
-   Hat die Anwendung sprachspezifische oder sprachunabhängige Ressourcen? Haben Sie beispielsweise die `UICulture`-Projekteigenschaft oder lokalisierbare Metadaten für `Application`-, `Page`- und `Resource`-Typen angegeben?  
  
-   Haben Sie die neutrale Spracheinstellung in der Datei „AssemblyInfo.cs“ oder „AssemblyInfo.vb“ verwendet?  
  
#### <a name="provide-scenario-and-impact-information"></a>Bereitstellen von Informationen zu Szenario und Auswirkungen  
 Informationen zum Szenario, in dem der Fehler auftritt, sowie die Auswirkungen des Fehlers sind äußerst wichtig für das [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)]-Team, um zu entscheiden, wann und wie ein Problem behoben werden sollte oder ob stattdessen eine akzeptable Umgehung des Problems verwendet werden könnte.  
  
 In der Regel haben mit Absturz und Datenverlust einhergehende Szenarien große Auswirkungen und können daher am leichtesten priorisiert werden. Einige Fehler treten jedoch nur in ungewöhnlichen Szenarien auf, die in einigen Fällen auch Hauptszenarien sein können. Die Bereitstellung von Kontext rund um das Szenario und die Auswirkungen unterstützen das [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)]-Team dabei, die richtige Entscheidung zu treffen.

<a name="MSDN_Community_Content"></a>   
## <a name="msdn-community-content"></a>MSDN Community Content  
 Die [Windows Presentation Foundation](../../../../docs/framework/wpf/index.md)-Dokumentation stellt umfassende technische Inhalte zu den Kernszenarien für [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)] bereit. Freilich treffen Entwickler in der realen Welt auf eine große Vielfalt von interessanten und nicht ganz üblichen Szenarien, aus denen sie Wichtiges lernen, das für die ganze [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)]-Entwicklercommunity nützlich sein kann.  
  
 Um Entwicklern die Weitergabe ihrer Kenntnisse und Erfahrungen an die [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)]-Community zu ermöglichen, wurde die Dokumentation nun durch den Bereich "Tonto's Expanding Head Band" ergänzt, in dem Entwickler Kommentare zu einzelnen Themen aus den Bereichen Konzepte, Referenzen, Gewusst wie-Themen, Tutorials und Beispiele hinzufügen und aktualisieren können.  
  
 Durch das Hinzufügen von Informationen zu "MSDN Community Content" wird ein zentraler, aktueller und gemeinsam genutzter Pool von [!INCLUDE[TLA2#tla_ms](../../../../includes/tla2sharptla-ms-md.md)]- und Community-Wissen geschaffen, der den Nutzen der Onlinedokumentation steigert. Darüber hinaus wird der Inhalt moderiert, um die Bereitstellung klarer, richtiger, geeigneter und nützlicher Informationen sicherzustellen.  
  
### <a name="what-type-of-content-should-be-added-to-msdn-community-content"></a>Welche Arten von Inhalten sollen MSDN Community Content hinzugefügt werden?  
 MSDN Community Content wurde entwickelt, um relevante, nützliche und angemessene Informationen zu erfassen, die die vorhandenen MSDN-Inhalte erweitern. Fehler, Feedback, Diskussionsthemen und Fragen sollten unter Verwendung der oben beschriebenen geeigneten und effektiven Tools eingebracht werden.  
  
### <a name="msdn-community-content-moderation"></a>Moderation von MSDN Community Content  
 MSDN Community Content wird moderiert, um die Angemessenheit der Inhalte sicherzustellen. Wie in den Richtlinien für die [Moderation von MSDN Community Content](http://go.microsoft.com/fwlink/?LinkId=98864) beschrieben, werden die folgenden Inhaltstypen als nicht geeignet betrachtet:  
  
-   Inhalte, die gegen die [Verhaltensregeln](http://go.microsoft.com/fwlink/?LinkId=98865) der MSDN Community verstoßen  
  
-   Irrelevante, unverständliche und Testbeiträge  
  
-   Doppelte Beiträge  
  
-   Spam  
  
-   Feedback zur Dokumentation  
  
-   Fehlerberichte  
  
-   Fragen  
  
 Moderatoren verarbeiten diese Inhalte gemäß den Richtlinien für die Moderation von MSDN Community Content.
