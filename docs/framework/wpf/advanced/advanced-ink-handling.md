# Erweiterte Behandlung von Freihandeingaben
Im Lieferumfang von [!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)] ist <xref:System.Windows.Controls.InkCanvas> enthalten. Dies ist ein Element zum Einbeziehen in eine Anwendung, um Freihandeingaben umgehend erfassen und anzeigen zu können.  Wenn das <xref:System.Windows.Controls.InkCanvas>\-Steuerelement jedoch keine ausreichende Steuerungstiefe bietet, kann durch Anpassen der eigenen Klassen zum Erfassen und Rendern von Freihandeingaben unter Verwendung von <xref:System.Windows.Input.StylusPlugIns> eine feinere Steuerung erreicht werden.  
  
 Die <xref:System.Windows.Input.StylusPlugIns>\-Klassen stellen einen Mechanismus zum Implementieren einer Low\-Level\-Steuerung von <xref:System.Windows.Input.Stylus>\-Eingaben und dynamischen Rendern von Freihandeingaben bereit.  Die <xref:System.Windows.Input.StylusPlugIns.StylusPlugIn>\-Klasse stellt Ihnen einen Mechanismus zum Implementieren von benutzerdefiniertem Verhalten bereit und wendet dieses für eine optimale Leistung auf den vom Tablettstift eingehenden Datenstream an.  Der <xref:System.Windows.Input.StylusPlugIns.DynamicRenderer>, ein spezialisiertes <xref:System.Windows.Input.StylusPlugIns.StylusPlugIn>, ermöglicht das dynamische Rendering von Freihanddaten in Echtzeit, d. h. der <xref:System.Windows.Input.StylusPlugIns.DynamicRenderer> zeichnet sofort digitale Freihandeingaben, während <xref:System.Windows.Input.StylusPoint>\-Daten generiert werden, sodass sie scheinbar aus dem Tablettstift fließen.
