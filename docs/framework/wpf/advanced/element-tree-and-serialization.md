# Elementstruktur und Serialisierung
Elemente der WPF\-Programmierung stehen häufig miteinander in einer strukturellen Beziehung.  Zum Beispiel kann eine in XAML erstellte Anwendungsbenutzeroberfläche als Objektstruktur angesehen werden.  Die Elementstruktur kann weiter in zwei diskrete Strukturen unterteilt werden, die jedoch auch parallel angeordnet sein können: die [logische Struktur](GTMT) und die [visuelle Struktur](GTMT).  Die Serialisierung in WPF umfasst das Speichern des Zustands dieser beiden Strukturen und des Zustands der Anwendung sowie das Schreiben in eine Datei, möglicherweise im XAML\-Format.
