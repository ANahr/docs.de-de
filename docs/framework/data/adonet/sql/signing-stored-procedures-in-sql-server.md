# <a name="signing-stored-procedures-in-sql-server"></a><span data-ttu-id="2a786-102">Signieren von gespeicherten Prozeduren in SQL Server</span><span class="sxs-lookup"><span data-stu-id="2a786-102">Signing Stored Procedures in SQL Server</span></span>
<span data-ttu-id="2a786-103">Sie können gespeicherte Prozeduren mit einem Zertifikat oder einem asymmetrischen Schlüssel signieren.</span><span class="sxs-lookup"><span data-stu-id="2a786-103">You can sign a stored procedure with a certificate or an asymmetric key.</span></span> <span data-ttu-id="2a786-104">Gedacht ist dies für Szenarien, in denen Berechtigungen nicht über die Besitzverkettung geerbt werden können oder in denen die Besitzkette unterbrochen ist, wie bei dynamischem SQL.</span><span class="sxs-lookup"><span data-stu-id="2a786-104">This is designed for scenarios when permissions cannot be inherited through ownership chaining or when the ownership chain is broken, such as dynamic SQL.</span></span> <span data-ttu-id="2a786-105">In einem solchen Fall erstellen Sie einen Benutzer, der dem Zertifikat zugeordnet wird, wodurch ihm Berechtigungen für die Objekte gewährt werden, auf die die gespeicherte Prozedur zugreifen können muss.</span><span class="sxs-lookup"><span data-stu-id="2a786-105">You then create a user mapped to the certificate, granting the certificate user permissions on the objects the stored procedure needs to access.</span></span>  
  
 <span data-ttu-id="2a786-106">Bei der Ausführung der gespeicherten Prozedur kombiniert SQL Server die Berechtigungen des Zertifikatsbenutzers mit den Berechtigungen des Aufrufers.</span><span class="sxs-lookup"><span data-stu-id="2a786-106">When the stored procedure is executed, SQL Server combines the permissions of the certificate user with those of the caller.</span></span> <span data-ttu-id="2a786-107">Im Gegensatz zur EXECUTE AS-Klausel ändert sich der Ausführungskontext der Prozedur nicht.</span><span class="sxs-lookup"><span data-stu-id="2a786-107">Unlike the EXECUTE AS clause, it does not change the execution context of the procedure.</span></span> <span data-ttu-id="2a786-108">Integrierte Funktionen, die Anmelde- und Benutzernamen zurückgeben, geben den Namen des Aufrufers, und nicht den Namen des Zertifikatsbenutzers zurück.</span><span class="sxs-lookup"><span data-stu-id="2a786-108">Built-in functions that return login and user names return the name of the caller, not the certificate user name.</span></span>  
  
 <span data-ttu-id="2a786-109">Eine digitale Signatur ist ein mit dem privaten Schlüssel des Signaturgebers verschlüsselter Datendigest.</span><span class="sxs-lookup"><span data-stu-id="2a786-109">A digital signature is a data digest encrypted with the private key of the signer.</span></span> <span data-ttu-id="2a786-110">Der private Schlüssel stellt sicher, dass die digitale Signatur für den Träger oder Besitzer eindeutig ist.</span><span class="sxs-lookup"><span data-stu-id="2a786-110">The private key ensures that the digital signature is unique to its bearer or owner.</span></span> <span data-ttu-id="2a786-111">Signiert werden können gespeicherte Prozeduren, Funktionen oder Trigger.</span><span class="sxs-lookup"><span data-stu-id="2a786-111">You can sign stored procedures, functions, or triggers.</span></span>  
  
> [!NOTE]
>  <span data-ttu-id="2a786-112">Sie können in der Masterdatenbank ein Zertifikat zum Gewähren von Berechtigungen auf Serverebene erstellen.</span><span class="sxs-lookup"><span data-stu-id="2a786-112">You can create a certificate in the master database to grant server-level permissions.</span></span>  
  
## <a name="creating-certificates"></a><span data-ttu-id="2a786-113">Erstellen von Zertifikaten</span><span class="sxs-lookup"><span data-stu-id="2a786-113">Creating Certificates</span></span>  
 <span data-ttu-id="2a786-114">Beim Signieren einer gespeicherten Prozedur mit einem Zertifikat wird mithilfe des privaten Schlüssels ein Datendigest erstellt, der aus dem verschlüsselten Hash der gespeicherten Prozedur besteht.</span><span class="sxs-lookup"><span data-stu-id="2a786-114">When you sign a stored procedure with a certificate, a data digest consisting of the encrypted hash of the stored procedure code is created using the private key.</span></span> <span data-ttu-id="2a786-115">Zur Laufzeit wird der Datendigest mit dem öffentlichen Schlüssel entschlüsselt und mit dem Hashwert der gespeicherten Prozedur verglichen.</span><span class="sxs-lookup"><span data-stu-id="2a786-115">At run time, the data digest is decrypted with the public key and compared with the hash value of the stored procedure.</span></span> <span data-ttu-id="2a786-116">Wenn die gespeicherte Prozedur geändert wird, wird der Hashwert ungültig, sodass die digitale Signatur nicht mehr übereinstimmt.</span><span class="sxs-lookup"><span data-stu-id="2a786-116">Modifying the stored procedure invalidates the hash value so that the digital signature no longer matches.</span></span> <span data-ttu-id="2a786-117">Auf diese Weise wird verhindert, dass der Code der gespeicherten Prozedur durch einen Benutzer geändert wird, der keine Zugriffsberechtigung für den privaten Schlüssel besitzt.</span><span class="sxs-lookup"><span data-stu-id="2a786-117">This prevents someone who does not have access to the private key from changing the stored procedure code.</span></span> <span data-ttu-id="2a786-118">Daher muss die Prozedur nach jeder Änderung neu signiert werden.</span><span class="sxs-lookup"><span data-stu-id="2a786-118">Therefore you must re-sign the procedure each time you modify it.</span></span>  
  
 <span data-ttu-id="2a786-119">Zum Signieren eines Moduls sind die folgenden vier Schritte auszuführen:</span><span class="sxs-lookup"><span data-stu-id="2a786-119">There are four steps involved in signing a module:</span></span>  
  
1.  <span data-ttu-id="2a786-120">Erstellen Sie mit der Transact-SQL-`CREATE CERTIFICATE [certificateName]`-Anweisung ein Zertifikat.</span><span class="sxs-lookup"><span data-stu-id="2a786-120">Create a certificate using the Transact-SQL `CREATE CERTIFICATE [certificateName]` statement.</span></span> <span data-ttu-id="2a786-121">Diese Anweisung verfügt über mehrere Optionen, mit denen das Start- und Enddatum und ein Kennwort festgelegt werden können.</span><span class="sxs-lookup"><span data-stu-id="2a786-121">This statement has several options for setting a start and end date and a password.</span></span> <span data-ttu-id="2a786-122">Die Standardgültigkeitsdauer beträgt ein Jahr.</span><span class="sxs-lookup"><span data-stu-id="2a786-122">The default expiration date is one year</span></span>  
  
2.  <span data-ttu-id="2a786-123">Erstellen Sie mit der Transact-SQL-`CREATE USER [userName] FROM CERTIFICATE [certificateName]`-Anweisung einen mit diesem Zertifikat verknüpften Datenbankbenutzer.</span><span class="sxs-lookup"><span data-stu-id="2a786-123">Create a database user associated with that certificate using the Transact-SQL `CREATE USER [userName] FROM CERTIFICATE [certificateName]` statement.</span></span> <span data-ttu-id="2a786-124">Dieser Benutzer ist nur in der Datenbank vorhanden, und er wird keiner Anmeldung zugeordnet.</span><span class="sxs-lookup"><span data-stu-id="2a786-124">This user exists in the database only and is not associated with a login.</span></span>  
  
3.  <span data-ttu-id="2a786-125">Gewähren Sie dem Zertifikatsbenutzer die erforderlichen Berechtigungen für die Datenbankobjekte.</span><span class="sxs-lookup"><span data-stu-id="2a786-125">Grant the certificate user the required permissions on the database objects.</span></span>  
  
> [!NOTE]
>  <span data-ttu-id="2a786-126">Ein Zertifikat kann keine Berechtigungen für Benutzer gewähren, die Berechtigungen hatten, die mit der DENY-Anweisung widerrufen wurden.</span><span class="sxs-lookup"><span data-stu-id="2a786-126">A certificate cannot grant permissions to a user that has had permissions revoked using the DENY statement.</span></span> <span data-ttu-id="2a786-127">DENY hat immer Vorrang gegenüber GRANT und verhindert, dass der Aufrufer Berechtigungen erben kann, die dem Zertifikatsbenutzer gewährt wurden.</span><span class="sxs-lookup"><span data-stu-id="2a786-127">DENY always takes precedence over GRANT, preventing the caller from inheriting permissions granted to the certificate user.</span></span>  
  
1.  <span data-ttu-id="2a786-128">Signieren Sie die Prozedur mit dem Zertifikat. Verwenden Sie dazu die Transact-SQL-`ADD SIGNATURE TO [procedureName] BY CERTIFICATE [certificateName]`-Anweisung.</span><span class="sxs-lookup"><span data-stu-id="2a786-128">Sign the procedure with the certificate using the Transact-SQL `ADD SIGNATURE TO [procedureName] BY CERTIFICATE [certificateName]` statement.</span></span>
