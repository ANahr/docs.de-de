### <a name="to-author-a-control"></a><span data-ttu-id="49982-113">So erstellen Sie ein Steuerelement</span><span class="sxs-lookup"><span data-stu-id="49982-113">To author a control</span></span>

    -   <span data-ttu-id="49982-116">Welche Art grafische Benutzeroberfläche benötigen Sie?</span><span class="sxs-lookup"><span data-stu-id="49982-116">What kind of graphical interface do you need?</span></span>  
  
    -   <span data-ttu-id="49982-117">Welche besonderen Benutzerinteraktionen werden von diesem Steuerelement behandelt?</span><span class="sxs-lookup"><span data-stu-id="49982-117">What specific user interactions will this control handle?</span></span>  
  
    -   <span data-ttu-id="49982-118">Wird die benötigte Funktionalität von vorhandenen Steuerelementen bereitgestellt?</span><span class="sxs-lookup"><span data-stu-id="49982-118">Is the functionality you need provided by any existing controls?</span></span>  
  
    -   <span data-ttu-id="49982-119">Erhalten Sie die Funktionalität, die Sie benötigen, durch Kombinieren mehrerer Windows Forms-Steuerelemente?</span><span class="sxs-lookup"><span data-stu-id="49982-119">Can you get the functionality you need by combining several Windows Forms controls?</span></span>  
  
2.  <span data-ttu-id="49982-120">Bestimmen Sie, wenn Sie ein Objektmodell für das Steuerelement benötigen, wie die Funktionalität im Objektmodell verteilt wird, und teilen Sie die Funktionalität zwischen dem Steuerelement und beliebigen Unterobjekten auf.</span><span class="sxs-lookup"><span data-stu-id="49982-120">If you need an object model for your control, determine how functionality will be distributed throughout the object model, and divide up functionality between the control and any subobjects.</span></span> <span data-ttu-id="49982-121">Ein Objektmodell kann nützlich sein, wenn Sie ein komplexes Steuerelement planen oder mehrere Funktionen eingebunden werden soll.</span><span class="sxs-lookup"><span data-stu-id="49982-121">An object model may be useful if you are planning a complex control, or want to incorporate several functionalities.</span></span>  
  
3.  <span data-ttu-id="49982-122">Bestimmen Sie, welchen Typ des Steuerelements (z.B. Benutzersteuerelement, benutzerdefiniertes Steuerelement, geerbtes Windows Forms-Steuerelement) Sie benötigen.</span><span class="sxs-lookup"><span data-stu-id="49982-122">Determine the type of control (for example, user control, custom control, inherited Windows Forms control) you need.</span></span> <span data-ttu-id="49982-123">Weitere Informationen finden Sie unter [Empfehlungen zum Typ von Steuerelementen](../../../../docs/framework/winforms/controls/control-type-recommendations.md) und [Varieties of Custom Controls (Vielfalt benutzerdefinierter Steuerelemente)](../../../../docs/framework/winforms/controls/varieties-of-custom-controls.md).</span><span class="sxs-lookup"><span data-stu-id="49982-123">For details, see [Control Type Recommendations](../../../../docs/framework/winforms/controls/control-type-recommendations.md) and [Varieties of Custom Controls](../../../../docs/framework/winforms/controls/varieties-of-custom-controls.md).</span></span>  
  
4.  <span data-ttu-id="49982-124">Drücken Sie die Funktionalität als Eigenschaften, Methoden und Ereignisse des Steuerelements und seiner Unterobjekte oder untergeordneten Strukturen aus, und weisen Sie entsprechende Zugriffsebenen zu (z.B. öffentlich, geschützt und so weiter).</span><span class="sxs-lookup"><span data-stu-id="49982-124">Express functionality as properties, methods, and events of the control and its subobjects or subsidiary structures, and assign appropriate access levels (for example, public, protected, and so on).</span></span>  
  
5.  <span data-ttu-id="49982-125">Wenn Sie eine benutzerdefinierte Darstellung Ihres Steuerelements benötigen, fügen Sie entsprechenden Code hinzu.</span><span class="sxs-lookup"><span data-stu-id="49982-125">If you need custom painting for your control, add code for it.</span></span> <span data-ttu-id="49982-126">Weitere Informationen finden Sie unter [Zeichnen und Ausgeben benutzerdefinierter Steuerelemente](../../../../docs/framework/winforms/controls/custom-control-painting-and-rendering.md).</span><span class="sxs-lookup"><span data-stu-id="49982-126">For details, see [Custom Control Painting and Rendering](../../../../docs/framework/winforms/controls/custom-control-painting-and-rendering.md).</span></span>  
  
6.  <span data-ttu-id="49982-127">Wenn das Steuerelement erbt <xref:System.Windows.Forms.UserControl>, Sie können das Laufzeitverhalten testen, indem Sie das Projekt erstellen und Ausführen den **UserControl-Testcontainer**.</span><span class="sxs-lookup"><span data-stu-id="49982-127">If your control inherits from <xref:System.Windows.Forms.UserControl>, you can test its runtime behavior by building the control project and running it in the **UserControl Test Container**.</span></span> <span data-ttu-id="49982-128">Weitere Informationen finden Sie unter [Vorgehensweise: Testen des Laufzeitverhaltens eines UserControl](../../../../docs/framework/winforms/controls/how-to-test-the-run-time-behavior-of-a-usercontrol.md).</span><span class="sxs-lookup"><span data-stu-id="49982-128">For more information, see [How to: Test the Run-Time Behavior of a UserControl](../../../../docs/framework/winforms/controls/how-to-test-the-run-time-behavior-of-a-usercontrol.md).</span></span>  
  
7.  <span data-ttu-id="49982-129">Sie können Ihr Steuerelement auch testen und debuggen, indem Sie ein neues Projekt erstellen, z.B. eine Windows-Anwendung, und es in einem Container platzieren.</span><span class="sxs-lookup"><span data-stu-id="49982-129">You can also test and debug your control by creating a new project, such as a Windows Application, and placing it into a container.</span></span> <span data-ttu-id="49982-130">Dieser Vorgang wird als Teil von [Exemplarische Vorgehensweise: Erstellen eines zusammengesetzten Steuerelements mit Visual Basic](../../../../docs/framework/winforms/controls/walkthrough-authoring-a-composite-control-with-visual-basic.md) veranschaulicht.</span><span class="sxs-lookup"><span data-stu-id="49982-130">This process is demonstrated as part of [Walkthrough: Authoring a Composite Control with Visual Basic](../../../../docs/framework/winforms/controls/walkthrough-authoring-a-composite-control-with-visual-basic.md).</span></span>  
  
8.  <span data-ttu-id="49982-131">Fügen Sie jedes Mal, wenn Sie die Funktionen hinzufügen, auch Funktionen zum Testprojekt hinzu, um die neue Funktionalität auszuführen.</span><span class="sxs-lookup"><span data-stu-id="49982-131">As you add each feature, add features to your test project to exercise the new functionality.</span></span>  
  
9. <span data-ttu-id="49982-132">Wiederholen Sie die Schritte, um den Entwurf zu verbessern.</span><span class="sxs-lookup"><span data-stu-id="49982-132">Repeat, refining the design.</span></span>  
  
10. <span data-ttu-id="49982-133">Verpacken Sie das Steuerelement, und stellen Sie es bereit.</span><span class="sxs-lookup"><span data-stu-id="49982-133">Package and deploy your control.</span></span> <span data-ttu-id="49982-134">Weitere Informationen finden Sie unter [Bereitstellen von Anwendungen, Diensten und Komponenten](https://msdn.microsoft.com/library/wtzawcsz).</span><span class="sxs-lookup"><span data-stu-id="49982-134">For details, see [Deploying Applications, Services, and Components](https://msdn.microsoft.com/library/wtzawcsz).</span></span>
