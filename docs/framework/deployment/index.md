# Bereitstellen von .NET Framework
Dieser Abschnitt der .NET Framework\-Dokumentation enthält Informationen für Entwickler, die .NET Framework mit ihren Anwendungen installieren möchten, und Administratoren, die .NET Framework über ein Netzwerk bereitstellen möchten.  Es werden außerdem Probleme mit der Aktivierung und dem Neustart erläutert, die im Zusammenhang mit der Bereitstellung auftreten können, sowie die Statusüberwachung Ihrer .NET Framework\-Installation.  
  
## In diesem Abschnitt  
 [Handbuch für die Bereitstellung für Entwickler](../../../docs/framework/deployment/deployment-guide-for-developers.md)  
 Es wird erklärt, wie Entwickler .NET Framework auf den Computern der Benutzer mit ihren Anwendungen installieren können.  
  
 [Handbuch für die Bereitstellung für Administratoren](../../../docs/framework/deployment/guide-for-administrators.md)  
 Hierin wird erläutert, wie ein Systemadministrator mit System Center Configuration Manager \(SCCM\) .NET Framework und dessen Systemabhängigkeiten in einem Netzwerk bereitstellen kann.  
  
 [Reduzieren von Systemneustarts bei .NET Framework 4.5\-Installationen](../../../docs/framework/deployment/reducing-system-restarts.md)  
 Beschreibt den Neustart\-Manager, der Neustarts nach Möglichkeit verhindert, und erläutert, wie Anwendungen, mit denen .NET Framework installiert wird, den Neustart\-Manager nutzen können.  
  
 [Gewusst wie: Abrufen des Status vom Installationsprogramm für .NET Framework 4.5](../../../docs/framework/deployment/how-to-get-progress-from-the-dotnet-installer.md)  
 Beschreibt, wie der .NET Framework\-Setupvorgang automatisch gestartet und nachverfolgt und dabei eine eigene Ansicht des Setupstatus angezeigt werden kann.  
  
 [.NET Framework\-Initialisierungsfehler: Verwalten der Benutzerfreundlichkeit](../../../docs/framework/deployment/initialization-errors-managing-the-user-experience.md)  
 Erklärt, was geschieht, wenn eine .NET Framework\-Anwendung eine CLR\-Version erfordert, die ungültig oder nicht auf dem Computer des Benutzers installiert ist, wie diese Fehler behoben werden können und wie die Fehlermeldung, die dem Benutzer angezeigt wird, gesteuert werden kann.  
  
 [Gewusst wie: Debuggen von CLR\-Aktivierungsproblemen](../../../docs/framework/deployment/how-to-debug-clr-activation-issues.md)  
 Erläutert, wie Sie CLR\-Aktivierungsprotokolle anzeigen und debuggen können, um Probleme zu beheben, die möglicherweise beim Ausführen der Anwendung mit der richtigen Version der CLR auftreten können.
