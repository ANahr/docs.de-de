# <a name="custom-tokens"></a>Benutzerdefinierte Token
Obwohl Windows Communication Foundation (WCF) systemintern x. 509-Zertifikate, Sicherheitskontexttoken, Kerberos-Tickets und Benutzernamentoken als Anmeldeinformationen unterstützt, ist es flexibel genug, damit Sie Ihre eigenen benutzerdefinierten Anmeldeinformationen verwenden können.  
  
## <a name="in-this-section"></a>In diesem Abschnitt  
 [Vorgehensweise: Erstellen eines benutzerdefinierten Tokens](../../../../docs/framework/wcf/extending/how-to-create-a-custom-token.md)  
 Beschreibt das Erstellen eines benutzerdefinierten Sicherheitstokens mithilfe der <xref:System.IdentityModel.Tokens.SecurityToken>-Klasse und seine Integrierung in einen benutzerdefinierten Sicherheitstokenanbieter und -authentifizierer.
