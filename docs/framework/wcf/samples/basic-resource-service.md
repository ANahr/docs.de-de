# <a name="basic-resource-service"></a><span data-ttu-id="c0a49-102">Einfacher Ressourcendienst</span><span class="sxs-lookup"><span data-stu-id="c0a49-102">Basic Resource Service</span></span>
<span data-ttu-id="c0a49-103">Dieses Beispiel veranschaulicht das Implementieren eines HTTP-basierter Diensts mithilfe der Windows Communication Foundation (WCF) REST-Programmiermodell, das eine Auflistung von Kunden verfügbar macht, die unterstützt das Abrufen, hinzufügen, löschen und Ersetzen-Vorgänge.</span><span class="sxs-lookup"><span data-stu-id="c0a49-103">This sample demonstrates how to implement a HTTP-based service using the Windows Communication Foundation (WCF) REST Programming model that exposes a collection of customers that supports the retrieve, add, delete and replace operations.</span></span> <span data-ttu-id="c0a49-104">In diesem Beispiel besteht aus 2 Komponenten – einen selbst gehosteten WCF-HTTP-Dienst (Service.cs) und eine Konsolenanwendung (program.cs), die der Dienst erstellt und Aufrufe durchführt.</span><span class="sxs-lookup"><span data-stu-id="c0a49-104">This sample consists of 2 components - a self-hosted WCF HTTP service (Service.cs) and a console application (program.cs) that creates the service and makes calls to it.</span></span>  
  
## <a name="sample-details"></a><span data-ttu-id="c0a49-105">Beispieldetails</span><span class="sxs-lookup"><span data-stu-id="c0a49-105">Sample Details</span></span>  
 <span data-ttu-id="c0a49-106">Der WCF-Dienst macht eine Auflistung von Kunden auf eine Ressource ressourcenorientierte/REST-Weise verfügbar.</span><span class="sxs-lookup"><span data-stu-id="c0a49-106">The WCF service exposes a collection of customers in a resource-oriented/REST manner.</span></span> <span data-ttu-id="c0a49-107">Kurz gesagt gehören dazu eindeutige URIs für die Auflistung der Kunden und für jeden Kunden in der Auflistung.</span><span class="sxs-lookup"><span data-stu-id="c0a49-107">In short, this involves having unique URIs for the collection of customers and every customer in the collection.</span></span> <span data-ttu-id="c0a49-108">Der Dienst unterstützt das Senden von HTTP `GET` am Auflistungs-URI, um die gesamte Auflistung abzurufen, und das Senden von HTTP `POST` am Auflistungs-URI, um der Auflistung einen neuen Kunden hinzuzufügen.</span><span class="sxs-lookup"><span data-stu-id="c0a49-108">The service supports sending an HTTP `GET` at the collection URI to retrieve the entire collection and HTTP `POST` at the collection URI to add a new customer to the collection.</span></span> <span data-ttu-id="c0a49-109">Der Dienst unterstützt am URI für einen einzelnen Kunden außerdem HTTP `GET`, um die Kundendetails abzurufen, HTTP `PUT`, um die Kundendetails zu ersetzen, und HTTP `DELETE`, um den Kunden aus der Auflistung zu entfernen.</span><span class="sxs-lookup"><span data-stu-id="c0a49-109">Also at the URI for an individual customer, it supports HTTP `GET` to get the customer details, HTTP `PUT` to replace the ihm einen eindeutigen URI zu und speichert den URI zusammen mit den anderen Kundendetails.</span><span class="sxs-lookup"><span data-stu-id="c0a49-110">When a new customer is added to the collection, the service assigns it a unique URI and stores the URI as part of the customer’s details.</span></span> <span data-ttu-id="c0a49-111">Der Dienst verwendet außerdem den HTTP-Speicherortheader der Antwort, um den URI dem Client mitzuteilen.</span><span class="sxs-lookup"><span data-stu-id="c0a49-111">Also, it communicates the URI to the client using the Location HTTP header of the response.</span></span>  
  
 <span data-ttu-id="c0a49-112">Die Datei App.config konfiguriert den WCF-Dienst mit einem Standard-<xref:System.ServiceModel.Description.WebHttpEndpoint>, für den die <xref:System.ServiceModel.Description.WebHttpEndpoint.HelpEnabled%2A>-Eigenschaft auf `true` festgelegt ist.</span><span class="sxs-lookup"><span data-stu-id="c0a49-112">The App.config file configures the WCF service with a default <xref:System.ServiceModel.Description.WebHttpEndpoint> that has the <xref:System.ServiceModel.Description.WebHttpEndpoint.HelpEnabled%2A> property set to `true`.</span></span> <span data-ttu-id="c0a49-113">Daher erstellt die WCF eine automatische HTML-basierte Hilfeseite unter `http://localhost:8000/Customers/help` , bereitstellt, finden Sie Informationen zum Erstellen von HTTP-Anforderungen für den Dienst und wie die HTTP-Antwort des Diensts – z. B. den Zugriff auf verdeutlicht, wie die Details des Kunden wird in XML und JSON dargestellt.</span><span class="sxs-lookup"><span data-stu-id="c0a49-113">As a result, the WCF creates an automatic HTML-based help page at `http://localhost:8000/Customers/help` that provides information about how to construct HTTP requests to the service and how to access the service’s HTTP response – for instance, an example of how the customer details is represented in XML and JSON.</span></span>  
  
 <span data-ttu-id="c0a49-114">Indem der Client die Kundenauflistung (und, allgemeiner ausgedrückt, jede Ressource) auf diese Weise verfügbar macht, kann er mithilfe von URIs sowie HTTP `GET`, `PUT`, `DELETE` und `POST` auf einheitliche Weise mit der Auflistung interagieren.</span><span class="sxs-lookup"><span data-stu-id="c0a49-114">Exposing the customer collection (and more generally, any resource) in this manner allows the client to interact with it in a uniform way using URIs and HTTP `GET`, `PUT`, `DELETE` and `POST`.</span></span> <span data-ttu-id="c0a49-115">Program.cs veranschaulicht, wie ein solcher Client mit <xref:System.Net.HttpWebRequest> erstellt werden kann.</span><span class="sxs-lookup"><span data-stu-id="c0a49-115">Program.cs demonstrates how such a client can be authored using <xref:System.Net.HttpWebRequest>.</span></span> <span data-ttu-id="c0a49-116">Beachten Sie, dass dies nur eine Möglichkeit, einen WCF REST-Dienst zuzugreifen.</span><span class="sxs-lookup"><span data-stu-id="c0a49-116">Note that this is just one way to access a WCF REST service.</span></span> <span data-ttu-id="c0a49-117">Es ist auch möglich, mit anderen .NET Framework-Klassen wie <xref:System.ServiceModel.ChannelFactory> und <xref:System.Net.WebClient> auf den Dienst zuzugreifen.</span><span class="sxs-lookup"><span data-stu-id="c0a49-117">It is also possible to access the service using other .NET Framework classes like the <xref:System.ServiceModel.ChannelFactory> and <xref:System.Net.WebClient>.</span></span> <span data-ttu-id="c0a49-118">In anderen Beispielen im SDK (z. B. die [grundlegenden HTTP-Dienst](../../../../docs/framework/wcf/samples/basic-http-service.md) Beispiel und die [automatische Formatauswahl](../../../../docs/framework/wcf/samples/automatic-format-selection.md) Beispiel) zeigen, wie Sie diese Klassen verwenden, um die Kommunikation mit einem WCF-Dienst.</span><span class="sxs-lookup"><span data-stu-id="c0a49-118">Other samples in the SDK (such as the [Basic HTTP Service](../../../../docs/framework/wcf/samples/basic-http-service.md) sample and the [Automatic Format Selection](../../../../docs/framework/wcf/samples/automatic-format-selection.md) sample) show how to use these classes to communicate with a WCF service.</span></span>  
  
 <span data-ttu-id="c0a49-119">Das Beispiel umfasst einen selbst gehosteten Dienst und einen Client, die in einer Konsolenanwendung ausgeführt werden.</span><span class="sxs-lookup"><span data-stu-id="c0a49-119">The sample consists a self-hosted service and a client that both run within a console application.</span></span> <span data-ttu-id="c0a49-120">Während die Konsolenanwendung ausgeführt wird, sendet der Client Anforderungen an den Dienst und schreibt die in den Antworten enthaltenen wichtigen Informationen in das Konsolenfenster.</span><span class="sxs-lookup"><span data-stu-id="c0a49-120">As the console application runs, the client makes requests to the service and writes the pertinent information from the responses to the console window.</span></span>
