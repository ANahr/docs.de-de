## <a name="sample-details"></a><span data-ttu-id="d7b27-105">Beispieldetails</span><span class="sxs-lookup"><span data-stu-id="d7b27-105">Sample Details</span></span>  
 <span data-ttu-id="d7b27-106">Das Beispiel besteht aus einem einfachen Dienst und Clientcode, mit dem Anforderungen an den Dienst gesendet werden.</span><span class="sxs-lookup"><span data-stu-id="d7b27-106">The sample consists of a simple service along with client code that makes requests to the service.</span></span>  <span data-ttu-id="d7b27-107">Der Dienst unterstützt einen einzelnen [WebGet-]Vorgang mit der folgenden Methodensignatur: `Message EchoListWithGet(string list);`</span><span class="sxs-lookup"><span data-stu-id="d7b27-107">The service supports a single [WebGet] operation, which has the following method signature: `Message EchoListWithGet(string list);`</span></span>  
  
 <span data-ttu-id="d7b27-108">Wenn der Client eine Anforderung an den Dienst sendet, stellt er eine durch Trennzeichen getrennte Liste der Elemente aus dem `list`-Abfragezeichenfolgenparameter bereit.  <span data-ttu-id="d7b27-109">Das vom Dienst zurückgegebene Antwortformat wird von einem `format`-Abfragezeichenfolgenparameter und zusätzlich von einem mit der Anforderung angegebenen HTTP Accept-Header bestimmt.</span><span class="sxs-lookup"><span data-stu-id="d7b27-109">The response format returned by the service is determined first by a `format` query-string parameter and second by an HTTP Accept header supplied with the request.</span></span> <span data-ttu-id="d7b27-110">Wenn der Wert des `format`-Abfragezeichenfolgenparameters einem der vorangehenden Formate entspricht, wird die Antwort im entsprechenden Format zurückgegeben.</span><span class="sxs-lookup"><span data-stu-id="d7b27-110">If the value of `format` query-string parameter is one of the preceding formats, then the response is returned in that format.</span></span> <span data-ttu-id="d7b27-111">Wenn die `format`-Abfragezeichenfolge nicht vorhanden ist, durchläuft der Dienst die Accept-Headerelemente der Anforderung und gibt das Format des ersten Inhaltstyp zurück, den der Dienst unterstützt.</span><span class="sxs-lookup"><span data-stu-id="d7b27-111">If the `format` query-string is not present, then the service iterates through the Accept header elements from the request and returns the format of the first content-type that the service supports.</span></span>  
  
 <span data-ttu-id="d7b27-112">Beim Rückgabetyp des Vorgangs ist Folgendes zu beachten.</span><span class="sxs-lookup"><span data-stu-id="d7b27-112">The return type of the operation is worth noting.</span></span> <span data-ttu-id="d7b27-113">Das [!INCLUDE[indigo2](../../../../includes/indigo2-md.md)]-REST-Programmiermodell unterstützt XML- und JSON-Antwortformate systemintern nur dann, wenn ein Vorgang einen anderen Typ als <xref:System.ServiceModel.Channels.Message> zurückgibt.</span><span class="sxs-lookup"><span data-stu-id="d7b27-113">The [!INCLUDE[indigo2](../../../../includes/indigo2-md.md)] REST programming model only natively supports XML and JSON response formats when an operation returns a type other than <xref:System.ServiceModel.Channels.Message>.</span></span> <span data-ttu-id="d7b27-114">Wenn als Rückgabetyp jedoch <xref:System.ServiceModel.Channels.Message> verwendet wird, kann der Entwickler vollständig selbst bestimmen, wie der Inhalt der Meldung formatiert werden soll.</span><span class="sxs-lookup"><span data-stu-id="d7b27-114">However, when using <xref:System.ServiceModel.Channels.Message> as the return type, the developer has complete control over how the content of the message should be formatted.</span></span>
