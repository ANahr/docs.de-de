# <a name="xml-literals-and-the-xml-10-specification-visual-basic"></a>XML-Literale und die XML 1.0-Spezifikation (Visual Basic)
Die Syntax der XML-Literale in Visual Basic unterstützt die meisten der Extensible Markup Language (XML) 1.0-Spezifikation. Ausführliche Informationen zu XML 1.0-Spezifikation finden Sie unter [Extensible Markup Language (XML) 1.0](http://go.microsoft.com/fwlink/?LinkId=73927) auf der W3C-Website.  
  
## <a name="what-visual-basic-does-not-support"></a>Nicht von Visual Basic unterstützt  
  
-   Ein XML-literal kann keiner Dokumenttypdefinition (DTD) enthalten.  
  
-   Ein XML-Dokumentliteral muss mit einer XML-Dokumentdeklaration beginnen.  
  
-   Ein XML-literal kann nicht mehr als 65.535 Zeichen in einer Zeile enthalten.  
  
-   XML-Namespacepräfixe, Elementnamen und Attributnamen können nicht mehr als 1.024 Zeichen enthalten.  
  
## <a name="extra-features-that-visual-basic-supports"></a>Zusätzliche Funktionen, die Visual Basic unterstützt  
  
-   Die eingebetteten Ausdruck-Syntax im Dokument und Elementliteralen zulässig ist kein gültiges XML.
