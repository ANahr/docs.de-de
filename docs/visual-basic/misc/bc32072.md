# <a name="cannot-implement-interface-39ltinterfacename1gt39-because-its-implementation-could-conflict-with-the-implementation-of-another-implemented-interface-39ltinterfacename2gt39-for-some-type-arguments"></a><span data-ttu-id="17296-102">Kann nicht implementieren Schnittstelle &#39; &lt;schnittstellenname1&gt;&#39; da ihre Implementierung einen Konflikt mit der ebenfalls implementierten Schnittstelle &#39; konnte&lt; schnittstellenname2&gt;&#39; bei manchen Typargumenten</span><span class="sxs-lookup"><span data-stu-id="17296-102">Cannot implement interface &#39;&lt;interfacename1&gt;&#39; because its implementation could conflict with the implementation of another implemented interface &#39;&lt;interfacename2&gt;&#39; for some type arguments</span></span>
<span data-ttu-id="17296-103">Eine Klassendeklaration enthält eine `Implements` -Anweisung, die zwei oder mehr Schnittstellen angibt, aber mindestens eine der Schnittstellen ist generisch, und zwei der Implementierungen könnten bei bestimmten Werten der Typargumente im Konflikt stehen.</span><span class="sxs-lookup"><span data-stu-id="17296-103">A class declaration includes an `Implements` statement that specifies two or more interfaces, but at least one of the interfaces is generic and two of the implementations could conflict for certain values of type arguments.</span></span>  
  
 <span data-ttu-id="17296-104">Dieser Fehler kann durch die folgenden Anweisungen generiert werden.</span><span class="sxs-lookup"><span data-stu-id="17296-104">The following statements can generate this error.</span></span>  
  
```  
Public Interface iFace1  
    Sub testSub(ByVal arg As String)  
End Interface  
Public Interface iFace2(Of t)  
    Sub testSub(ByVal arg As t)  
End Interface  
Public Class testClass  
    Implements iFace1, iFace2(Of String)  
End Class  
```  
  
 <span data-ttu-id="17296-105">Da `iFace2` mithilfe von `String`konstruiert wird, muss `testClass` zwei Versionen von `testSub` mit identischen Signaturen implementieren.</span><span class="sxs-lookup"><span data-stu-id="17296-105">Because `iFace2` is constructed using `String`, `testClass` must implement two versions of `testSub` with identical signatures.</span></span> <span data-ttu-id="17296-106">Dies führt zu einer Mehrdeutigkeit in Bezug auf die Version, auf die zugegriffen werden soll.</span><span class="sxs-lookup"><span data-stu-id="17296-106">Doing so would produce an ambiguity about which version to access.</span></span>  
  
 <span data-ttu-id="17296-107">**Fehler-ID:** BC32072</span><span class="sxs-lookup"><span data-stu-id="17296-107">**Error ID:** BC32072</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="17296-108">So beheben Sie diesen Fehler</span><span class="sxs-lookup"><span data-stu-id="17296-108">To correct this error</span></span>  
  
-   <span data-ttu-id="17296-109">Ändern Sie das der generischen Schnittstelle übergebene Typargument so, dass kein Konflikt auftritt.</span><span class="sxs-lookup"><span data-stu-id="17296-109">Change the type argument supplied to the generic interface so that there is no conflict.</span></span>  
  
     <span data-ttu-id="17296-110">- oder -</span><span class="sxs-lookup"><span data-stu-id="17296-110">-or-</span></span>  
  
-   <span data-ttu-id="17296-111">Entfernen Sie die `Implements` -Anweisung aus einer der Schnittstellen, die zum Konflikt bei der Implementierung führen.</span><span class="sxs-lookup"><span data-stu-id="17296-111">Remove from the `Implements` statement one of the interfaces resulting in the implementation conflict.</span></span>
