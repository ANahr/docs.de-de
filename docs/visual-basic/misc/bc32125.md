# <a name="cannot-implement-39ltinterfacename1gtltmembernamegt39-because-its-implementation-could-conflict-with-the-implementation-for-39ltinterfacename2gtltmembernamegt39-for-some-type-arguments"></a><span data-ttu-id="8ec77-102">Kann nicht implementieren &#39; &lt;schnittstellenname1&gt;.&lt; Membername&gt;&#39; da ihre Implementierung mit der Implementierung für &#39; einen Konflikt würde&lt; schnittstellenname2&gt;.&lt; Membername&gt;&#39; bei manchen Typargumenten</span><span class="sxs-lookup"><span data-stu-id="8ec77-102">Cannot implement &#39;&lt;interfacename1&gt;.&lt;membername&gt;&#39; because its implementation could conflict with the implementation for &#39;&lt;interfacename2&gt;.&lt;membername&gt;&#39; for some type arguments</span></span>
<span data-ttu-id="8ec77-103">Eine Klasse implementiert mehrere generische Schnittstellen, von denen eine von einer anderen erbt, und zwei Implementierungen eines Schnittstellenmembers können einen Konflikt mit bestimmten Werten von Typargumenten verursachen.</span><span class="sxs-lookup"><span data-stu-id="8ec77-103">A class implements more than one generic interface, one of which inherits from another, and two implementations of an interface member could conflict for certain values of type arguments.</span></span>  
  
 <span data-ttu-id="8ec77-104">Dieser Fehler kann durch die folgenden Anweisungen generiert werden.</span><span class="sxs-lookup"><span data-stu-id="8ec77-104">The following statements can generate this error.</span></span>  
  
```  
Public Interface iFace1(Of t)  
    Sub testSub()  
End Interface  
Public Interface iFace2(Of u)  
    Inherits iFace1(Of u)  
End Interface  
Public Class testClass(Of y, z)  
    Implements iFace1(Of y), iFace2(Of z)  
    Public Sub testSuby() Implements iFace1(Of y).testSub  
    End Sub  
    Public Sub testSubz() Implements iFace1(Of z).testSub  
    End Sub  
End Class  
```  
  
 <span data-ttu-id="8ec77-105">Da `iFace2` von `iFace1` erbt (unter Verwendung des eigenen Typparameters`u`), würde `testClass` zwei Versionen von `iFace1.testSub` mit identischen Signaturen implementieren, wenn dasselbe Typargument an `y` und `z`übergeben würde.</span><span class="sxs-lookup"><span data-stu-id="8ec77-105">Because `iFace2` inherits from `iFace1` using its own type parameter (`u`), `testClass` would implement two versions of `iFace1.testSub` with identical signatures if the same type argument were passed to `y` and `z`.</span></span> <span data-ttu-id="8ec77-106">Dies würde zu einer Mehrdeutigkeit in Bezug auf die Version führen, auf die zugegriffen werden soll.</span><span class="sxs-lookup"><span data-stu-id="8ec77-106">This would produce an ambiguity about which version to access.</span></span>  
  
 <span data-ttu-id="8ec77-107">**Fehler-ID:** BC32125</span><span class="sxs-lookup"><span data-stu-id="8ec77-107">**Error ID:** BC32125</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="8ec77-108">So beheben Sie diesen Fehler</span><span class="sxs-lookup"><span data-stu-id="8ec77-108">To correct this error</span></span>  
  
-   <span data-ttu-id="8ec77-109">Ändern Sie die Vererbungsstruktur der Schnittstellen so, dass `iFace1` nicht mit zwei verschiedenen Typargumenten implementiert werden kann.</span><span class="sxs-lookup"><span data-stu-id="8ec77-109">Change the inheritance structure of the interfaces so that `iFace1` could not be implemented with two different type arguments.</span></span>  
  
     <span data-ttu-id="8ec77-110">- oder - </span><span class="sxs-lookup"><span data-stu-id="8ec77-110">-or-</span></span>  
  
-   <span data-ttu-id="8ec77-111">Entfernen Sie die `Implements` -Anweisung aus einer der Schnittstellen, die zum Konflikt bei der Implementierung führen.</span><span class="sxs-lookup"><span data-stu-id="8ec77-111">Remove from the `Implements` statement one of the interfaces resulting in the implementation conflict.</span></span>
