# <a name="type-argument-inference-failed-for-type-parameter-39lttypeparametername1gt39-of-39ltgenericproceduresignaturegt39"></a><span data-ttu-id="7f974-102">Argument Typrückschluss konnte nicht für den Typparameter &#39; &lt;typparametername1&gt; &#39; von &#39; &lt;Signatur&gt;&#39;</span><span class="sxs-lookup"><span data-stu-id="7f974-102">Type argument inference failed for type parameter &#39;&lt;typeparametername1&gt;&#39; of &#39;&lt;genericproceduresignature&gt;&#39;</span></span>
<span data-ttu-id="7f974-103">Argument Typrückschluss konnte nicht für den Typparameter "\<typparametername1 >' von '\<Signatur >".</span><span class="sxs-lookup"><span data-stu-id="7f974-103">Type argument inference failed for type parameter '\<typeparametername1>' of '\<genericproceduresignature>'.</span></span> <span data-ttu-id="7f974-104">Ein Typargument, das per Rückschluss von dem an den Parameter übergeben "\<parametername1 >' steht in Konflikt mit dem Typargument, das per Rückschluss von dem an den Parameter übergeben"\<parametername2 > ".</span><span class="sxs-lookup"><span data-stu-id="7f974-104">Type argument inferred from the argument passed to parameter '\<parametername1>' conflicts with the type argument inferred from the argument passed to parameter '\<parametername2>'.</span></span>  
  
 <span data-ttu-id="7f974-105">Eine generische Prozedur wird ohne Typargumente aufgerufen, und durch den Versuch, den Typ abzuleiten, wurde eine Datentypkonflikt zwischen den Typparametern verursacht.</span><span class="sxs-lookup"><span data-stu-id="7f974-105">A generic procedure is called without any type arguments, and the attempted type inference has produced a data type conflict among the type parameters.</span></span>  
  
 <span data-ttu-id="7f974-106">Wenn Sie eine generische Prozedur aufrufen, geben Sie in der Regel für jeden Typparameter, der durch die generische Prozedur definiert wird, ein Typargument an.</span><span class="sxs-lookup"><span data-stu-id="7f974-106">Normally, when you call a generic procedure, you supply a type argument for each type parameter that the generic procedure defines.</span></span> <span data-ttu-id="7f974-107">Wenn Sie keine Typargumente angeben, versucht der Compiler, die an die Typparameter zu übergebenden Typen abzuleiten.</span><span class="sxs-lookup"><span data-stu-id="7f974-107">If you do not supply any type arguments, then the compiler attempts to infer the types to be passed to the type parameters.</span></span> <span data-ttu-id="7f974-108">Wenn durch den Kontext des Aufrufs Datentypinformationen für einen Typparameter bereitgestellt werden, die einen Konflikt verursachen, schlägt der Typrückschluss fehl.</span><span class="sxs-lookup"><span data-stu-id="7f974-108">If the context of the call provides conflicting data type information for a type parameter, then the type inference fails.</span></span>  
  
 <span data-ttu-id="7f974-109">Dieser Fehler kann durch folgenden Code generiert werden.</span><span class="sxs-lookup"><span data-stu-id="7f974-109">The following code can generate this error.</span></span>  
  
```  
Public Sub takeTwoValues(Of t)(ByVal x As t, ByVal y As t)  
End Sub  
Call takeTwoValues(4, 2.5)  
```  
  
 <span data-ttu-id="7f974-110">Weil der Compiler aufgrund des ersten Arguments `Integer` für den Typparameter `t`ableitet, aber aufgrund des zweiten Arguments für denselben Typparameter `Double` ableitet, entsteht ein Konflikt in Bezug auf den an `t`zu übergebenden Datentyp.</span><span class="sxs-lookup"><span data-stu-id="7f974-110">Because the first argument causes the compiler to infer `Integer` for type parameter `t`, while the second argument causes it to infer `Double` for the same type parameter, there is a conflict regarding which data type to pass to `t`.</span></span>  
  
 <span data-ttu-id="7f974-111">**Fehler-ID:** BC32116</span><span class="sxs-lookup"><span data-stu-id="7f974-111">**Error ID:** BC32116</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="7f974-112">So beheben Sie diesen Fehler</span><span class="sxs-lookup"><span data-stu-id="7f974-112">To correct this error</span></span>  
  
-   <span data-ttu-id="7f974-113">Geben Sie Typargumente für den generischen Typ an, damit der Compiler sie nicht ableiten muss.</span><span class="sxs-lookup"><span data-stu-id="7f974-113">Supply type arguments to the generic type, so that the compiler does not have to infer them.</span></span>  
  
    ```  
    Call takeTwoValues(Of Double)(4, 2.5)  
    ```  
  
     <span data-ttu-id="7f974-114">Beachten Sie, dass in diesem Fall, in dem zwei normale Argumente unterschiedliche Datentypen aufweisen, der aufrufende Code ein Typargument übergeben muss, das den Anforderungen dieser beiden Datentypen entspricht.</span><span class="sxs-lookup"><span data-stu-id="7f974-114">Notice that in this case, where the two normal arguments are of different data types, the calling code must pass a type argument that can accommodate both of those data types.</span></span> <span data-ttu-id="7f974-115">In diesem Fall wird `Integer` zu `Double`erweitert.</span><span class="sxs-lookup"><span data-stu-id="7f974-115">In this case, `Integer` widens to `Double`.</span></span>  
  
     <span data-ttu-id="7f974-116">- oder - </span><span class="sxs-lookup"><span data-stu-id="7f974-116">-or-</span></span>  
  
-   <span data-ttu-id="7f974-117">Definieren Sie die generische Prozedur neu, indem Sie für die normalen Parameter unterschiedliche Typparameter angeben, sodass beim Ableiten der Typen kein Konflikt entsteht.</span><span class="sxs-lookup"><span data-stu-id="7f974-117">Redefine the generic procedure to specify different type parameters for the normal parameters, so that there is no conflict inferring the types.</span></span>  
  
    ```  
    Public Sub takeTwoValues(Of t1, t2)(ByVal x As t1, ByVal y As t2)  
    ```
