## <a name="the-regular-expressions-cache"></a><span data-ttu-id="afad4-119">Der Cache für reguläre Ausdrücke</span><span class="sxs-lookup"><span data-stu-id="afad4-119">The Regular Expressions Cache</span></span>  
 <span data-ttu-id="afad4-120">Zur Verbesserung der Leistung verwaltet das Modul für reguläre Ausdrücke einen anwendungsweiten Cache kompilierter regulärer Ausdrücke.</span><span class="sxs-lookup"><span data-stu-id="afad4-120">To improve performance, the regular expression engine maintains an application-wide cache of compiled regular expressions.</span></span> <span data-ttu-id="afad4-121">Der Cache speichert Muster für reguläre Ausdrücke, die nur in statischen Methodenaufrufen verwendet werden.</span><span class="sxs-lookup"><span data-stu-id="afad4-121">The cache stores regular expression patterns that are used only in static method calls.</span></span> <span data-ttu-id="afad4-122">(An Instanzmethoden übergebene Muster für reguläre Ausdrücke werden nicht zwischengespeichert.) Dadurch wird vermieden, dass Ausdrücke bei jeder Verwendung erneut analysiert und in Bytecode höherer Ebene kompiliert werden müssen.</span><span class="sxs-lookup"><span data-stu-id="afad4-122">(Regular expression patterns supplied to instance methods are not cached.) This avoids the need to reparse an expression into high-level byte code each time it is used.</span></span>  
  
 <span data-ttu-id="afad4-123">Die maximale Anzahl der zwischengespeicherten regulären Ausdrücke richtet sich nach dem Wert der `static` (`Shared` in Visual Basic) <xref:System.Text.RegularExpressions.Regex.CacheSize%2A?displayProperty=nameWithType> Eigenschaft.</span><span class="sxs-lookup"><span data-stu-id="afad4-123">The maximum number of cached regular expressions is determined by the value of the `static` (`Shared` in Visual Basic) <xref:System.Text.RegularExpressions.Regex.CacheSize%2A?displayProperty=nameWithType> property.</span></span> <span data-ttu-id="afad4-124">Standardmäßig speichert das Modul für reguläre Ausdrücke bis zu 15 kompilierte reguläre Ausdrücke zwischen.</span><span class="sxs-lookup"><span data-stu-id="afad4-124">By default, the regular expression engine caches up to 15 compiled regular expressions.</span></span> <span data-ttu-id="afad4-125">Wenn die Anzahl der kompilierten regulären Ausdrücke die Cachegröße überschreitet, wird der am längsten nicht verwendete reguläre Ausdruck verworfen und der neue reguläre Ausdruck zwischengespeichert.</span><span class="sxs-lookup"><span data-stu-id="afad4-125">If the number of compiled regular expressions exceeds the cache size, the least recently used regular expression is discarded and the new regular expression is cached.</span></span>  
  
 <span data-ttu-id="afad4-126">Es gibt zwei Möglichkeiten, wie die Anwendung vorkompilierte reguläre Ausdrücke nutzen kann:</span><span class="sxs-lookup"><span data-stu-id="afad4-126">Your application can take advantage of precompiled regular expressions in one of the following two ways:</span></span>  
  
-   <span data-ttu-id="afad4-127">Eine statische Methode mit der <xref:System.Text.RegularExpressions.Regex> Objekt, das den regulären Ausdruck definieren.</span><span class="sxs-lookup"><span data-stu-id="afad4-127">By using a static method of the <xref:System.Text.RegularExpressions.Regex> object to define the regular expression.</span></span> <span data-ttu-id="afad4-128">Wenn Sie ein Muster für reguläre Ausdrücke verwenden, das bereits in einem anderen statischen Methodenaufruf definiert wurde, ruft das Modul für reguläre Ausdrücke dieses aus dem Cache ab.</span><span class="sxs-lookup"><span data-stu-id="afad4-128">If you are using a regular expression pattern that has already been defined in another static method call, the regular expression engine will retrieve it from the cache.</span></span> <span data-ttu-id="afad4-129">Ist dies nicht der Fall, kompiliert das Modul den regulären Ausdruck und fügt ihn dem Cache hinzu.</span><span class="sxs-lookup"><span data-stu-id="afad4-129">If not, the engine will compile the regular expression and add it to the cache.</span></span>  
  
-   <span data-ttu-id="afad4-130">Durch das Wiederverwenden einer vorhandenen <xref:System.Text.RegularExpressions.Regex> Objekt wie entsprechende Muster des regulären Ausdrucks benötigt wird.</span><span class="sxs-lookup"><span data-stu-id="afad4-130">By reusing an existing <xref:System.Text.RegularExpressions.Regex> object as long as its regular expression pattern is needed.</span></span>  
  
 <span data-ttu-id="afad4-131">Wegen des Aufwandes der Objektinstanziierung und Kompilierung regulärer Ausdrücke, erstellen und Zerstören von schnell zahlreiche <xref:System.Text.RegularExpressions.Regex> Objekte ist ein sehr aufwändiger Vorgang.</span><span class="sxs-lookup"><span data-stu-id="afad4-131">Because of the overhead of object instantiation and regular expression compilation, creating and rapidly destroying numerous <xref:System.Text.RegularExpressions.Regex> objects is a very expensive process.</span></span> <span data-ttu-id="afad4-132">Für Anwendungen, die eine große Anzahl von anderen regulären Ausdrücken verwendet, können Sie Optimieren der Leistung mithilfe der Aufrufe von statischen `Regex` Methoden und erhöhen die Größe des Caches für reguläre Ausdrücke durch.</span><span class="sxs-lookup"><span data-stu-id="afad4-132">For applications that use a large number of different regular expressions, you can optimize performance by using calls to static `Regex` methods and possibly by increasing the size of the regular expression cache.</span></span>
