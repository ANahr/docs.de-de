# <a name="docker-terminology"></a><span data-ttu-id="2f763-104">Docker-Terminologie</span><span class="sxs-lookup"><span data-stu-id="2f763-104">Docker terminology</span></span>

<span data-ttu-id="2f763-105">In diesem Abschnitt werden die Begriffe und Definitionen aufgelistet, mit denen Sie vertraut sein sollten, bevor Sie sich ausführlicher mit Docker beschäftigen.</span><span class="sxs-lookup"><span data-stu-id="2f763-105">This section lists terms and definitions you should be familiar with before getting deeper into Docker.</span></span> <span data-ttu-id="2f763-106">Weitere Definitionen finden Sie im umfangreichen, von Docker bereitgestellten [Glossar](https://docs.docker.com/glossary/).</span><span class="sxs-lookup"><span data-stu-id="2f763-106">For further definitions, see the extensive [glossary](https://docs.docker.com/glossary/) provided by Docker .</span></span>

<span data-ttu-id="2f763-107">**Containerimage**: Ein Paket mit allen benötigten Abhängigkeiten und Informationen zum Erstellen eines Containers.</span><span class="sxs-lookup"><span data-stu-id="2f763-107">**Container image**: A package with all the dependencies and information needed to create a container.</span></span> <span data-ttu-id="2f763-108">Ein Image enthält alle Abhängigkeiten (z.B. Frameworks) sowie die Bereitstellung und Ausführung der Konfiguration, die von einer Containerruntime verwendet werden.</span><span class="sxs-lookup"><span data-stu-id="2f763-108">An image includes all the dependencies (such as frameworks) plus deployment and execution configuration to be used by a container runtime.</span></span> <span data-ttu-id="2f763-109">In der Regel wird ein Image aus mehreren zugrundeliegenden Images abgeleitet, die übereinander liegen und das Dateisystem des Containers bilden.</span><span class="sxs-lookup"><span data-stu-id="2f763-109">Usually, an image derives from multiple base images that are layers stacked on top of each other to form the container’s filesystem.</span></span> <span data-ttu-id="2f763-110">Ein Image ist unveränderlich, nachdem es erstellt wurde.</span><span class="sxs-lookup"><span data-stu-id="2f763-110">An image is immutable once it has been created.</span></span>

<span data-ttu-id="2f763-111">**Container**: Eine Instanz eines Docker-Images.</span><span class="sxs-lookup"><span data-stu-id="2f763-111">**Container**: An instance of a Docker image.</span></span> <span data-ttu-id="2f763-112">Ein Container stellt die Ausführung einer einzelnen Anwendung, eines Prozesses oder Diensts dar.</span><span class="sxs-lookup"><span data-stu-id="2f763-112">A container represents the execution of a single application, process, or service.</span></span> <span data-ttu-id="2f763-113">Er besteht aus den Inhalten eines Docker-Images, einer Ausführungsumgebung und mehreren Standardanweisungen.</span><span class="sxs-lookup"><span data-stu-id="2f763-113">It consists of the contents of a Docker image, an execution environment, and a standard set of instructions.</span></span> <span data-ttu-id="2f763-114">Beim Skalieren eines Diensts erstellen Sie mehrere Instanzen eines Containers aus dem gleichen Image.</span><span class="sxs-lookup"><span data-stu-id="2f763-114">When scaling a service, you create multiple instances of a container from the same image.</span></span> <span data-ttu-id="2f763-115">Alternativ kann ein Batchauftrag mehrere Container aus dem gleichen Image erstellen und dabei verschiedene Parameter an jede Instanz übergeben.</span><span class="sxs-lookup"><span data-stu-id="2f763-115">Or a batch job can create multiple containers from the same image, passing different parameters to each instance.</span></span>

<span data-ttu-id="2f763-116">**Tag**: Eine Markierung oder eine Bezeichnung, die Sie auf Images anwenden können, damit verschiedene Images oder Versionen desselben Images (abhängig von der Versionsnummer oder der Zielumgebung) identifiziert werden können.</span><span class="sxs-lookup"><span data-stu-id="2f763-116">**Tag**: A mark or label you can apply to images so that different images or versions of the same image (depending on the version number or the target environment) can be identified.</span></span>

<span data-ttu-id="2f763-117">**Docker-Datei**: Eine Textdatei, die Anweisungen zum Erstellen eines Docker-Images enthält.</span><span class="sxs-lookup"><span data-stu-id="2f763-117">**Dockerfile**: A text file that contains instructions for how to build a Docker image.</span></span>

<span data-ttu-id="2f763-118">**Build**: Die Aktion zum Erstellen eines Containerimages auf der Grundlage der Informationen und des Kontexts der Docker-Datei sowie weiteren Dateien im Ordner, in dem das Image erstellt wird.</span><span class="sxs-lookup"><span data-stu-id="2f763-118">**Build**: The action of building a container image based on the information and context provided by its Dockerfile, plus additional files in the folder where the image is built.</span></span> <span data-ttu-id="2f763-119">Sie können Images mit dem Docker-Befehl „docker build“ erstellen.</span><span class="sxs-lookup"><span data-stu-id="2f763-119">You can build images with the Docker docker build command.</span></span>

<span data-ttu-id="2f763-120">**Repository**: Eine Auflistung von zugehörigen Docker-Images mit dem Tag, das die Version des Images angibt.</span><span class="sxs-lookup"><span data-stu-id="2f763-120">**Repository (repo)**: A collection of related Docker images, labeled with a tag that indicates the image version.</span></span> <span data-ttu-id="2f763-121">Einige Repositorys enthalten mehrere Varianten eines bestimmtes Images, z.B. ein Image mit SDKs (schwerer), ein Image nur mit Runtimes (leichter), usw. Diese Varianten können mit Tags markiert werden.</span><span class="sxs-lookup"><span data-stu-id="2f763-121">Some repos contain multiple variants of a specific image, such as an image containing SDKs (heavier), an image containing only runtimes (lighter), etc. Those variants can be marked with tags.</span></span> <span data-ttu-id="2f763-122">Ein einzelnes Repository kann Plattformvarianten enthalten, z.B. ein Linux-Image und ein Windows-Image.</span><span class="sxs-lookup"><span data-stu-id="2f763-122">A single repo can contain platform variants, such as a Linux image and a Windows image.</span></span>

<span data-ttu-id="2f763-123">**Registrierung**: Ein Dienst, der Zugriff auf Repositorys bereitstellt.</span><span class="sxs-lookup"><span data-stu-id="2f763-123">**Registry**: A service that provides access to repositories.</span></span> <span data-ttu-id="2f763-124">Die Standardregistrierung für die meisten öffentlichen Images ist der [Docker-Hub](https://hub.docker.com/) (im Besitz des Docker-Unternehmens).</span><span class="sxs-lookup"><span data-stu-id="2f763-124">The default registry for most public images is [Docker Hub](https://hub.docker.com/) (owned by Docker as an organization).</span></span> <span data-ttu-id="2f763-125">Eine Registrierung enthält in der Regel Repositorys mehrerer Teams.</span><span class="sxs-lookup"><span data-stu-id="2f763-125">A registry usually contains repositories from multiple teams.</span></span> <span data-ttu-id="2f763-126">Unternehmen haben häufig private Registrierungen zum Speichern und Verwalten von Images, die sie erstellt haben.</span><span class="sxs-lookup"><span data-stu-id="2f763-126">Companies often have private registries to store and manage images they’ve created.</span></span> <span data-ttu-id="2f763-127">Azure Container Registry ist ein weiteres Beispiel.</span><span class="sxs-lookup"><span data-stu-id="2f763-127">Azure Container Registry is another example.</span></span>

<span data-ttu-id="2f763-128">**Docker-Hub**: Eine öffentliche Registrierung zum Hochladen von und Arbeiten mit Images.</span><span class="sxs-lookup"><span data-stu-id="2f763-128">**Docker Hub**: A public registry to upload images and work with them.</span></span> <span data-ttu-id="2f763-129">Docker-Hub bietet Docker-Imagehosts, öffentliche oder private Registrierungen, Buildtrigger, Web-Hooks und Integration mit GitHub und Bitbucket.</span><span class="sxs-lookup"><span data-stu-id="2f763-129">Docker Hub provides Docker image hosting, public or private registries, build triggers and web hooks, and integration with GitHub and Bitbucket.</span></span>

<span data-ttu-id="2f763-130">**Azure Container Registry**: Eine öffentliche Ressource zur Arbeit mit Docker-Images und deren Komponenten in Azure.</span><span class="sxs-lookup"><span data-stu-id="2f763-130">**Azure Container Registry**: A public resource for working with Docker images and its components in Azure.</span></span> <span data-ttu-id="2f763-131">Dies stellt eine Registrierung in der Nähe Ihrer Bereitstellungen in Azure zur Verfügung. Sie erhalten somit Kontrolle über den Zugriff und können Ihre Azure Active Directory-Gruppen und -Berechtigungen verwenden.</span><span class="sxs-lookup"><span data-stu-id="2f763-131">This provides a registry that is close to your deployments in Azure and that gives you control over access, making it possible to use your Azure Active Directory groups and permissions.</span></span>

<span data-ttu-id="2f763-132">**Docker Trusted Registry (DTR)**: Ein Docker-Registrierungsdienst (von Docker), der lokal installiert werden kann, sodass er innerhalb des Datencenters und Netzwerks des Unternehmens aktiv ist.</span><span class="sxs-lookup"><span data-stu-id="2f763-132">**Docker Trusted Registry (DTR)**: A Docker registry service (from Docker) that can be installed on-premises so it lives within the organization’s datacenter and network.</span></span> <span data-ttu-id="2f763-133">Er ist praktisch für private Images, die innerhalb des Unternehmens verwaltet werden sollen.</span><span class="sxs-lookup"><span data-stu-id="2f763-133">It is convenient for private images that should be managed within the enterprise.</span></span> <span data-ttu-id="2f763-134">Docker Trusted Registry ist als Bestandteil im Produkt Docker Datacenter enthalten.</span><span class="sxs-lookup"><span data-stu-id="2f763-134">Docker Trusted Registry is included as part of the Docker Datacenter product.</span></span> <span data-ttu-id="2f763-135">Weitere Informationen finden Sie unter [Docker Trusted Registry (DTR)](https://docs.docker.com/docker-trusted-registry/overview/).</span><span class="sxs-lookup"><span data-stu-id="2f763-135">For more information, see [Docker Trusted Registry (DTR)](https://docs.docker.com/docker-trusted-registry/overview/).</span></span>

<span data-ttu-id="2f763-136">**Docker Community Edition (CE)**: Entwicklungstools für Windows und macOS zum lokalen Erstellen, Ausführen und Testen von Containern.</span><span class="sxs-lookup"><span data-stu-id="2f763-136">**Docker Community Edition (CE)**: Development tools for Windows and macOS for building, running, and testing containers locally.</span></span> <span data-ttu-id="2f763-137">Docker CE für Windows bietet Entwicklungsumgebungen für Linux- und Windows-Container.</span><span class="sxs-lookup"><span data-stu-id="2f763-137">Docker CE for Windows provides development environments for both Linux and Windows Containers.</span></span> <span data-ttu-id="2f763-138">Der Linux-Docker-Host auf Windows basiert auf einer [Hyper-V](https://www.microsoft.com/en-us/server-cloud/solutions/virtualization.aspx)-VM.</span><span class="sxs-lookup"><span data-stu-id="2f763-138">The Linux Docker host on Windows is based on a [Hyper-V](https://www.microsoft.com/en-us/server-cloud/solutions/virtualization.aspx) virtual machine.</span></span> <span data-ttu-id="2f763-139">Der Host für die Windows-Container basiert direkt auf Windows.</span><span class="sxs-lookup"><span data-stu-id="2f763-139">The host for Windows Containers is directly based on Windows.</span></span> <span data-ttu-id="2f763-140">Docker CE für Mac basiert auf dem Apple Hypervisor-Framework und dem [Xhyve-Hypervisor](https://github.com/mist64/xhyve), der eine Linux-Docker-Host-VM auf Mac OS X bereitstellt. Docker CE für Windows und für Mac ersetzt Docker Toolbox, das auf Oracle VirtualBox basiert.</span><span class="sxs-lookup"><span data-stu-id="2f763-140">Docker CE for Mac is based on the Apple Hypervisor framework and the [xhyve hypervisor](https://github.com/mist64/xhyve), which provides a Linux Docker host virtual machine on Mac OS X. Docker CE for Windows and for Mac replaces Docker Toolbox, which was based on Oracle VirtualBox.</span></span>

<span data-ttu-id="2f763-141">**Docker Enterprise Edition (EE)**: Eine Version der Docker-Tools für Unternehmen für die Linux- und Windows-Entwicklung.</span><span class="sxs-lookup"><span data-stu-id="2f763-141">**Docker Enterprise Edition (EE)**: An enterprise-scale version of Docker tools for Linux and Windows development.</span></span>

<span data-ttu-id="2f763-142">**Compose**: Ein Befehlszeilentool und YAML-Dateiformat mit Metadaten zum Definieren und Ausführen von Anwendungen mit mehreren Container.</span><span class="sxs-lookup"><span data-stu-id="2f763-142">**Compose**: A command-line tool and YAML file format with metadata for defining and running multi-container applications.</span></span> <span data-ttu-id="2f763-143">Sie definieren eine einzelne Anwendung basierend auf mehreren Images mit einer oder mehreren .yml-Dateien, die Werte abhängig von der Umgebung überschreiben können.</span><span class="sxs-lookup"><span data-stu-id="2f763-143">You define a single application based on multiple images with one or more .yml files that can override values depending on the environment.</span></span> <span data-ttu-id="2f763-144">Nachdem Sie die Definitionen erstellt haben, können Sie die gesamte Anwendung mit mehreren Containern mit einem einzigen Befehl (docker-compose up) bereitstellen, der einen Container pro Image auf dem Docker-Host erstellt.</span><span class="sxs-lookup"><span data-stu-id="2f763-144">After you have created the definitions, you can deploy the whole multi-container application with a single command (docker-compose up) that creates a container per image on the Docker host.</span></span>

<span data-ttu-id="2f763-145">**Cluster**: Eine Sammlung von zur Verfügung gestellten Docker-Hosts, als handele es sich um einen einzelnen, virtuellen Docker-Host, sodass die Anwendung mehrere Instanzen der Dienste skalieren kann, die auf mehrere Hosts im Cluster verteilt sind.</span><span class="sxs-lookup"><span data-stu-id="2f763-145">**Cluster**: A collection of Docker hosts exposed as if it were a single virtual Docker host, so that the application can scale to multiple instances of the services spread across multiple hosts within the cluster.</span></span> <span data-ttu-id="2f763-146">Docker-Cluster können mit Docker Swarm, Mesosphere DC/OS, Kubernetes und Azure Service Fabric erstellt werden.</span><span class="sxs-lookup"><span data-stu-id="2f763-146">Docker clusters can be created with Docker Swarm, Mesosphere DC/OS, Kubernetes, and Azure Service Fabric.</span></span> <span data-ttu-id="2f763-147">(Wenn Sie Docker Swarm zur Verwaltung eines Clusters verwenden, verweisen Sie in der Regel auf das Cluster als *Schwarm* anstelle eines Clusters.)</span><span class="sxs-lookup"><span data-stu-id="2f763-147">(If you use Docker Swarm for managing a cluster, you typically refer to the cluster as a *swarm* instead of a cluster.)</span></span>

<span data-ttu-id="2f763-148">**Orchestrator**: Ein Tool, das die Verwaltung von Clustern und Docker-Hosts vereinfacht.</span><span class="sxs-lookup"><span data-stu-id="2f763-148">**Orchestrator**: A tool that simplifies management of clusters and Docker hosts.</span></span> <span data-ttu-id="2f763-149">Orchestratoren ermöglichen es Ihnen, Ihre Images, Container und Hosts über eine Befehlszeilenschnittstelle (CLI) oder eine grafische Benutzeroberfläche zu verwalten.</span><span class="sxs-lookup"><span data-stu-id="2f763-149">Orchestrators enable you to manage their images, containers, and hosts through a command line interface (CLI) or a graphical UI.</span></span> <span data-ttu-id="2f763-150">Sie können u.a. Containernetzwerke, Konfigurationen, den Lastenausgleich, die Dienstermittlung, die Hochverfügbarkeit und die Konfiguration des Docker-Hosts verwalten.</span><span class="sxs-lookup"><span data-stu-id="2f763-150">You can manage container networking, configurations, load balancing, service discovery, high availability, Docker host configuration, and more.</span></span> <span data-ttu-id="2f763-151">Ein Orchestrator ist verantwortlich für die Ausführung, Verteilung, Skalierung und Reparatur von Workloads in einer Knotensammlung.</span><span class="sxs-lookup"><span data-stu-id="2f763-151">An orchestrator is responsible for running, distributing, scaling, and healing workloads across a collection of nodes.</span></span> <span data-ttu-id="2f763-152">Orchestratorprodukte sind in der Regel die gleichen Produkte, die Clusterinfrastruktur wie Mesosphere DC/OS, Kubernetes, Docker Swarm und Azure Service Fabric bereitstellen.</span><span class="sxs-lookup"><span data-stu-id="2f763-152">Typically, orchestrator products are the same products that provide cluster infrastructure, like Mesosphere DC/OS, Kubernetes, Docker Swarm, and Azure Service Fabric.</span></span>
