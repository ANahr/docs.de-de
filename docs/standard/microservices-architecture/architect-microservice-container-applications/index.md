<span data-ttu-id="3e279-105">In diesem Leitfaden haben Sie bereits die grundlegenden Konzepte für Container und Docker gelernt.</span><span class="sxs-lookup"><span data-stu-id="3e279-105">Earlier in this guide, you learned basic concepts about containers and Docker.</span></span> <span data-ttu-id="3e279-106">Dabei handelt es sich um die mindestens erforderlichen Informationen, die Sie für die ersten Schritte mit Containern benötigen.</span><span class="sxs-lookup"><span data-stu-id="3e279-106">That was the minimum information you need in order to get started with containers.</span></span> <span data-ttu-id="3e279-107">Aber obwohl Container hilfreich und besonders für Microservices geeignet sind, sind sie für eine Microservice-Architektur nicht erforderlich. ## <a name="container-design-principles"></a><span data-ttu-id="3e279-112">Richtlinien für das Containerdesign</span><span class="sxs-lookup"><span data-stu-id="3e279-112">Container design principles</span></span>

<span data-ttu-id="3e279-113">Im Containermodell stellt eine Instanz eines Containerimages einen einzelnen Prozess dar.</span><span class="sxs-lookup"><span data-stu-id="3e279-113">In the container model, a container image instance represents a single process.</span></span> <span data-ttu-id="3e279-114">Indem ein Containerimage als Prozessgrenze definiert wird, können Sie Grundtypen erstellen, die zum Skalieren oder Stapeln des Prozesses verwendet werden können.</span><span class="sxs-lookup"><span data-stu-id="3e279-114">By defining a container image as a process boundary, you can create primitives that can be used to scale the process or to batch it.</span></span>

<span data-ttu-id="3e279-115">Wenn Sie ein Containerimage entworfen haben, wird Ihnen die Definition [ENTRYPOINT](https://docs.docker.com/engine/reference/builder/) in der Dockerfile angezeigt.</span><span class="sxs-lookup"><span data-stu-id="3e279-115">When you design a container image, you will see an [ENTRYPOINT](https://docs.docker.com/engine/reference/builder/) definition in the Dockerfile.</span></span> <span data-ttu-id="3e279-116">Dadurch wird der Prozess definiert, dessen Lebensdauer die Lebensdauer des Containers steuert.</span><span class="sxs-lookup"><span data-stu-id="3e279-116">This defines the process whose lifetime controls the lifetime of the container.</span></span> <span data-ttu-id="3e279-117">Wenn der Prozess abgeschlossen ist, endet der Lebenszyklus des Containers.</span><span class="sxs-lookup"><span data-stu-id="3e279-117">When the process completes, the container lifecycle ends.</span></span> <span data-ttu-id="3e279-118">Container können langlebige Prozesse wie Webserver, aber auch kurzlebige Prozesse wie Batchaufträge (früher als Azure [WebJobs](https://docs.microsoft.com/azure/app-service-web/websites-webjobs-resources) implementiert) darstellen.</span><span class="sxs-lookup"><span data-stu-id="3e279-118">Containers might represent long-running processes like web servers, but can also represent short-lived processes like batch jobs, which formerly might have been implemented as Azure [WebJobs](https://docs.microsoft.com/azure/app-service-web/websites-webjobs-resources).</span></span>

<span data-ttu-id="3e279-119">Wenn der Prozess fehlschlägt, wird der Container angehalten und der Orchestrator übernimmt seinen Platz.</span><span class="sxs-lookup"><span data-stu-id="3e279-119">If the process fails, the container ends, and the orchestrator takes over.</span></span> <span data-ttu-id="3e279-120">Wenn der Orchestrator dafür konfiguriert wurde, fünf Instanzen auszuführen und eine davon fehlschlägt, erstellt der Orchestrator eine andere Containerinstanz, um den fehlgeschlagenen Prozess zu ersetzen.</span><span class="sxs-lookup"><span data-stu-id="3e279-120">If the orchestrator was configured to keep five instances running and one fails, the orchestrator will create another container instance to replace the failed process.</span></span> <span data-ttu-id="3e279-121">In einem Batchauftrag wird der Prozess mit Parametern gestartet.</span><span class="sxs-lookup"><span data-stu-id="3e279-121">In a batch job, the process is started with parameters.</span></span> <span data-ttu-id="3e279-122">Wenn der Prozess abgeschlossen ist, ist die Arbeit abgeschlossen.</span><span class="sxs-lookup"><span data-stu-id="3e279-122">When the process completes, the work is complete.</span></span> <span data-ttu-id="3e279-123">Dieser Leitfaden geht zu einem späteren Zeitpunkt ausführlich auf Orchestratoren ein.</span><span class="sxs-lookup"><span data-stu-id="3e279-123">This guidance drills-down on orchestrators, later on.</span></span>

<span data-ttu-id="3e279-124">Es gibt Szenarios, in denen mehrere Prozesse in einem einzigen Container ausgeführt werden sollen.</span><span class="sxs-lookup"><span data-stu-id="3e279-124">You might find a scenario where you want multiple processes running in a single container.</span></span> <span data-ttu-id="3e279-125">Da es nur einen Einstiegspunkt pro Container geben kann, können Sie für dieses Szenario ein Skript innerhalb des Containers ausführen, das so viele Programme wie benötigt startet.</span><span class="sxs-lookup"><span data-stu-id="3e279-125">For that scenario, since there can be only one entry point per container, you could run a script within the container that launches as many programs as needed.</span></span> <span data-ttu-id="3e279-126">Sie können beispielsweise [Supervisor](http://supervisord.org/) oder ein ähnliches Tool verwenden, um mehrere Prozesse innerhalb eines einzigen Containers zu starten.</span><span class="sxs-lookup"><span data-stu-id="3e279-126">For example, you can use [Supervisor](http://supervisord.org/) or a similar tool to take care of launching multiple processes inside a single container.</span></span> <span data-ttu-id="3e279-127">Obwohl es Architekturen gibt, die mehrere Prozesse pro Container enthalten können, ist dieser Ansatz nicht üblich.</span><span class="sxs-lookup"><span data-stu-id="3e279-127">However, even though you can find architectures that hold multiple processes per container, that approach is not very common.</span></span>
