# <a name="microservices-architecture"></a><span data-ttu-id="b8ae4-104">Microservicearchitektur</span><span class="sxs-lookup"><span data-stu-id="b8ae4-104">Microservices architecture</span></span>

<span data-ttu-id="b8ae4-105">Wie der Name schon sagt, ist eine Microservices Architektur ein Ansatz zum Erstellen einer Server-Anwendung als eine Reihe von kleinen Diensten.</span><span class="sxs-lookup"><span data-stu-id="b8ae4-105">As the name implies, a microservices architecture is an approach to building a server application as a set of small services.</span></span> <span data-ttu-id="b8ae4-106">Jeder Dienst in einem eigenen Prozess ausgeführt wird und mit anderen Prozessen, die über Protokolle wie HTTP/HTTPS, WebSockets kommuniziert oder [AMQP](https://en.wikipedia.org/wiki/Advanced_Message_Queuing_Protocol).</span><span class="sxs-lookup"><span data-stu-id="b8ae4-106">Each service runs in its own process and communicates with other processes using protocols such as HTTP/HTTPS, WebSockets, or [AMQP](https://en.wikipedia.org/wiki/Advanced_Message_Queuing_Protocol).</span></span> <span data-ttu-id="b8ae4-107">Jedes Microservice implementiert wird, eine bestimmte End-to-End-Domäne oder Business-Funktion in einem bestimmten Kontext befindet, und jede muss autonom entwickelt und bereitgestellt werden, unabhängig.</span><span class="sxs-lookup"><span data-stu-id="b8ae4-107">Each microservice implements a specific end-to-end domain or business capability within a certain context boundary, and each must be developed autonomously and be deployable independently.</span></span> <span data-ttu-id="b8ae4-108">Zum Schluss sollte jede Microservice der verknüpften Domäne-Datenmodell und die Domänenlogik (Hoheit und dezentrale Data Management), die basierend auf verschiedene datenspeicherungstechnologien (SQL oder NoSQL) und anderen Programmiersprachen besitzen.</span><span class="sxs-lookup"><span data-stu-id="b8ae4-108">Finally, each microservice should own its related domain data model and domain logic (sovereignty and decentralized data management) based on different data storage technologies (SQL, NoSQL) and different programming languages.</span></span>

<span data-ttu-id="b8ae4-109">Welche Größe sollte ein Microservice werden?</span><span class="sxs-lookup"><span data-stu-id="b8ae4-109">What size should a microservice be?</span></span> <span data-ttu-id="b8ae4-110">Beim Entwickeln einer Microservice sollte Größe nicht wichtig.</span><span class="sxs-lookup"><span data-stu-id="b8ae4-110">When developing a microservice, size should not be the important point.</span></span> <span data-ttu-id="b8ae4-111">Stattdessen sollte der wichtige Punkt werden Services erstellen lose gekoppelt, daher müssen Sie die Autonomie der Entwicklung, Bereitstellung und Skalierung, für jeden Dienst.</span><span class="sxs-lookup"><span data-stu-id="b8ae4-111">Instead, the important point should be to create loosely coupled services so you have autonomy of development, deployment, and scale, for each service.</span></span> <span data-ttu-id="b8ae4-112">Natürlich beim Identifizieren und Microservices entwerfen, sollten Sie so zu machen, so klein wie möglich, solange Sie nicht zu viele direkte Abhängigkeiten mit anderen Microservices verfügen.</span><span class="sxs-lookup"><span data-stu-id="b8ae4-112">Of course, when identifying and designing microservices, you should try to make them as small as possible as long as you do not have too many direct dependencies with other microservices.</span></span> <span data-ttu-id="b8ae4-113">Wichtiger ist als die Größe der Microservice ist die interne Kohäsion, die sie benötigen und die Unabhängigkeit von anderen Diensten.</span><span class="sxs-lookup"><span data-stu-id="b8ae4-113">More important than the size of the microservice is the internal cohesion it must have and its independence from other services.</span></span>

<span data-ttu-id="b8ae4-114">Warum eine Microservices Architektur?</span><span class="sxs-lookup"><span data-stu-id="b8ae4-114">Why a microservices architecture?</span></span> <span data-ttu-id="b8ae4-115">Kurz gesagt, stellt er langfristigen Flexibilität bereit.</span><span class="sxs-lookup"><span data-stu-id="b8ae4-115">In short, it provides long-term agility.</span></span> <span data-ttu-id="b8ae4-116">Microservices aktivieren besseren Verwaltbarkeit in komplexen, große und höchst flexibel skalierbaren Systemen, da Sie das Erstellen von Anwendungen, die basierend auf viele unabhängig bereitstellbare Diensten, die eine präzise und autonome Lebenszyklen aufweisen können.</span><span class="sxs-lookup"><span data-stu-id="b8ae4-116">Microservices enable better maintainability in complex, large, and highly-scalable systems by letting you create applications based on many independently deployable services that each have granular and autonomous lifecycles.</span></span>

<span data-ttu-id="b8ae4-117">Ein weiterer Vorteil kann Microservices unabhängig voneinander skalieren.</span><span class="sxs-lookup"><span data-stu-id="b8ae4-117">As an additional benefit, microservices can scale out independently.</span></span> <span data-ttu-id="b8ae4-118">Anstatt eine einzelne monolithische Anwendung, die Sie als eine Einheit skalieren müssen, können Sie stattdessen bestimmte Microservices skalieren.</span><span class="sxs-lookup"><span data-stu-id="b8ae4-118">Instead of having a single monolithic application that you must scale out as a unit, you can instead scale out specific microservices.</span></span> <span data-ttu-id="b8ae4-119">Auf diese Weise können Sie nur den Funktionsbereich "skalieren, die weitere Verarbeitung Power oder die Netzwerkbandbreite zur Unterstützung bei Bedarf, anstatt die Skalierung anderen Bereichen der Anwendung, die nicht skaliert werden müssen.</span><span class="sxs-lookup"><span data-stu-id="b8ae4-119">That way, you can scale just the functional area that needs more processing power or network bandwidth to support demand, rather than scaling out other areas of the application that do not need to be scaled.</span></span> <span data-ttu-id="b8ae4-120">Das bedeutet Kosten einsparen, da Sie weniger Hardware erforderlich.</span><span class="sxs-lookup"><span data-stu-id="b8ae4-120">That means cost savings because you need less hardware.</span></span>

-   <span data-ttu-id="b8ae4-129">Überwachung und Integrität überprüft der Dienste und -Infrastruktur.</span><span class="sxs-lookup"><span data-stu-id="b8ae4-129">Monitoring and health checks of the services and infrastructure.</span></span>

-   <span data-ttu-id="b8ae4-130">Skalierbare Infrastruktur für die Dienste (d. h. Cloud und Orchestrators).</span><span class="sxs-lookup"><span data-stu-id="b8ae4-130">Scalable infrastructure for the services (that is, cloud and orchestrators).</span></span>

-   <span data-ttu-id="b8ae4-131">Sicherheit-Design und Implementierung auf mehreren Ebenen: Authentifizierung, Autorisierung, geheime Schlüssel Management, sichere Kommunikation usw..</span><span class="sxs-lookup"><span data-stu-id="b8ae4-131">Security design and implementation at multiple levels: authentication, authorization, secrets management, secure communication, etc.</span></span>

-   <span data-ttu-id="b8ae4-132">Schnelle Anwendung Übermittlung in der Regel mit verschiedenen Teams, die verschiedene Microservices konzentrieren.</span><span class="sxs-lookup"><span data-stu-id="b8ae4-132">Rapid application delivery, usually with different teams focusing on different microservices.</span></span>

-   <span data-ttu-id="b8ae4-133">DevOps und CI-CD-Methoden und -Infrastruktur.</span><span class="sxs-lookup"><span data-stu-id="b8ae4-133">DevOps and CI/CD practices and infrastructure.</span></span>
