<span data-ttu-id="495c0-105">In den meisten Fällen können Sie sich einen Container als Instanz eines Prozesses vorstellen.</span><span class="sxs-lookup"><span data-stu-id="495c0-105">In most cases, you can think of a container as an instance of a process.</span></span> <span data-ttu-id="495c0-106">Ein Prozess behält keinen persistenten Zustand bei.</span><span class="sxs-lookup"><span data-stu-id="495c0-106">A process does not maintain persistent state.</span></span> <span data-ttu-id="495c0-107">Während ein Container in den lokalen Speicher schreiben kann, gleicht die Annahme, dass eine Instanz auf unbestimmte Zeit vorhanden ist, der Annahme, dass ein Speicherort im Arbeitsspeicher dauerhaft ist.</span><span class="sxs-lookup"><span data-stu-id="495c0-107">While a container can write to its local storage, assuming that an instance will be around indefinitely would be like assuming that a single location in memory will be durable.</span></span> <span data-ttu-id="495c0-108">Für Containerimages, z.B. Prozesse, sollte angenommen werden, dass sie über mehrere Instanzen verfügen oder irgendwann abgebrochen werden. <span data-ttu-id="495c0-109">Docker bietet ein Feature mit dem Namen *Overlay File System* (Dateisystem überlagern).</span><span class="sxs-lookup"><span data-stu-id="495c0-109">Docker provides a feature named the *overlay file system*.</span></span> <span data-ttu-id="495c0-110">Dadurch wird ein copy-on-write-Task (Bei Schreibvorgang kopieren) implementiert, die aktualisierte Informationen in das Stammdateisystem des Containers speichert.</span><span class="sxs-lookup"><span data-stu-id="495c0-110">This implements a copy-on-write task that stores updated information to the root file system of the container.</span></span> <span data-ttu-id="495c0-111">Diese Informationen kommen zum ursprünglichen Image hinzu, auf dem der Container basiert.</span><span class="sxs-lookup"><span data-stu-id="495c0-111">That information is in addition to the original image on which the container is based.</span></span> <span data-ttu-id="495c0-112">Wenn der Container aus dem System gelöscht wird, gehen diese Änderungen verloren.</span><span class="sxs-lookup"><span data-stu-id="495c0-112">If the container is deleted from the system, those changes are lost.</span></span> <span data-ttu-id="495c0-113">Aus diesem Grund ist es zwar möglich, den Zustand eines Containers in seinem lokalen Speicher zu speichern, doch das Entwerfen eines umgehenden Systems würde mit der Prämisse des Containerdesigns kollidieren. Dieses Design ist standardmäßig zustandslos.</span><span class="sxs-lookup"><span data-stu-id="495c0-113">Therefore, while it is possible to save the state of a container within its local storage, designing a system around this would conflict with the premise of container design, which by default is stateless.</span></span>

-   <span data-ttu-id="495c0-117">[Volume-Plug-Ins](https://docs.docker.com/engine/tutorials/dockervolumes/), die Volumes in Remotedienste einbinden und so langfristige Persistenz bereitstellen.</span><span class="sxs-lookup"><span data-stu-id="495c0-117">[Volume plugins](https://docs.docker.com/engine/tutorials/dockervolumes/) that mount volumes to remote services, providing long-term persistence.</span></span>

-   <span data-ttu-id="495c0-119">Relationale Remotedatenbanken wie die [Azure SQL-Datenbank](https://azure.microsoft.com/services/sql-database/) oder NoSQL-Datenbanken wie [Azure Cosmos DB](https://docs.microsoft.com/azure/cosmos-db/introduction) oder Zwischenspeicherdienste wie [Redis](https://redis.io/).</span><span class="sxs-lookup"><span data-stu-id="495c0-119">Remote relational databases like [Azure SQL Database](https://azure.microsoft.com/services/sql-database/) or NoSQL databases like [Azure Cosmos DB](https://docs.microsoft.com/azure/cosmos-db/introduction), or cache services like [Redis](https://redis.io/).</span></span>

<span data-ttu-id="495c0-120">Nachfolgend erhalten Sie weitere Details zu diesen Optionen.</span><span class="sxs-lookup"><span data-stu-id="495c0-120">The following provides more detail about these options.</span></span>

<span data-ttu-id="495c0-121">**Datenvolumes** sind Verzeichnisse, die vom Hostbetriebssystem den Verzeichnissen in Containern zugeordnet werden.</span><span class="sxs-lookup"><span zuordnet.</span><span class="sxs-lookup"><span data-stu-id="495c0-123">This directory is not tied to the lifetime of the container itself, and the directory can be accessed from code running directly
