<span data-ttu-id="0ef4a-106">Als Stabilität wird die Fähigkeit zum Wiederherstellen nach Fehlern und zum Fortsetzen der Funktionsweise bezeichnet.</span><span class="sxs-lookup"><span data-stu-id="0ef4a-106">Resiliency is the ability to recover from failures and continue to function.</span></span> <span data-ttu-id="0ef4a-107">Stabilität ist nicht zum Vermeiden von Fehlern, jedoch akzeptiert die Tatsache, dass Fehler auftreten und reagiert dann in einer Weise, die Downtime und Datenverluste vermieden werden.</span><span class="sxs-lookup"><span data-stu-id="0ef4a-107">Resiliency is not about avoiding failures, but accepting the fact that failures will occur, and then responding to them in a way that avoids downtime or data loss.</span></span> <span data-ttu-id="0ef4a-108">Das Ziel der Stabilität ist, die Anwendung nach einem Fehler wieder in einen voll funktionsfähigen Zustand zu versetzen.</span><span class="sxs-lookup"><span data-stu-id="0ef4a-108">The goal of resiliency is to return the application to a fully functioning state after a failure.</span></span>

<span data-ttu-id="0ef4a-109">Die Anwendung ist für die Cloud bereit, wenn mindestens ein Modell softwarebasierten resilienzgrad, anstatt eine Hardware-basiertes Modell implementiert.</span><span class="sxs-lookup"><span data-stu-id="0ef4a-109">Your application is ready for the cloud when, at a minimum, it implements a software-based model of resiliency, rather than a hardware-based model.</span></span> <span data-ttu-id="0ef4a-110">Ihre Cloud-Anwendung muss die teilfehler nutzen, die sicherlich stattfinden.</span><span class="sxs-lookup"><span data-stu-id="0ef4a-110">Your cloud application must embrace the partial failures that will certainly occur.</span></span> <span data-ttu-id="0ef4a-111">Entwerfen Sie oder gestalten Sie teilweise die Anwendung mit erwarteten teilfehlern Stabilität zu erzielen.</span><span class="sxs-lookup"><span data-stu-id="0ef4a-111">Design or partially refactor your application to achieve resiliency with expected partial failures.</span></span> <span data-ttu-id="0ef4a-112">Es sollten so entworfen werden, zu bewältigen teilfehler, z. B. vorübergehende Netzwerkausfälle und Knoten oder virtuellen Computern in der Cloud abstürzt.</span><span class="sxs-lookup"><span data-stu-id="0ef4a-112">It should be designed to cope with partial failures, like transient network outages and nodes, or VMs crashing in the cloud.</span></span> <span data-ttu-id="0ef4a-113">Sogar Container verschoben werden, auf einen anderen Knoten innerhalb eines Clusters Orchestrator können kurze Ausfälle in der Anwendung verursachen.</span><span class="sxs-lookup"><span data-stu-id="0ef4a-113">Even containers being moved to a different node within an orchestrator cluster can cause intermittent short failures within the application.</span></span>

<span data-ttu-id="0ef4a-122">Ein ähnliches Szenario kann in einer app auftreten, die HTTP-Dienste verwendet.</span><span class="sxs-lookup"><span data-stu-id="0ef4a-122">A similar scenario might occur in an app that consumes HTTP services.</span></span> <span data-ttu-id="0ef4a-123">Das Netzwerk oder der Dienst selbst möglicherweise nicht in der Cloud während einer kurzen, vorübergehende Fehler verfügbar.</span><span class="sxs-lookup"><span data-stu-id="0ef4a-123">The network or the service itself might not be available in the cloud during a short, transient failure.</span></span>

<span data-ttu-id="0ef4a-124">Eine robuste Anwendung wie in Abbildung 4-9 sollten Techniken, wie "Wiederholungen mit exponenzieller" implementieren, so erteilen Sie der Anwendung Gelegenheit, vorübergehende Fehler in der Ressourcen zu behandeln.</span><span class="sxs-lookup"><span data-stu-id="0ef4a-124">A resilient application like the one shown in Figure 4-9 should implement techniques like "retries with exponential backoff" to give the application an opportunity to handle transient failures in resources.</span></span> <span data-ttu-id="0ef4a-125">Sie sollten auch "Trennschalter" in Ihren Anwendungen verwenden.</span><span class="sxs-lookup"><span data-stu-id="0ef4a-125">You also should use "circuit breakers" in your applications.</span></span> <span data-ttu-id="0ef4a-126">Ein Trennschalter wird eine Anwendung versucht, eine Ressource zuzugreifen, wenn es tatsächlich einen langfristigen Fehler wird beendet.</span><span class="sxs-lookup"><span data-stu-id="0ef4a-126">A circuit breaker stops an application from trying to access a resource when it's actually a long-term failure.</span></span> <span data-ttu-id="0ef4a-127">Mithilfe von ein Trennschalter vermeidet die Anwendung provozieren einen Denial of Service auf sich selbst.</span><span class="sxs-lookup"><span data-stu-id="0ef4a-127">By using a circuit breaker, the application avoids provoking a denial of service to itself.</span></span>
