# <a name="when-to-deploy-windows-containers-in-your-on-premises-iaas-vm-infrastructure"></a>Wenn für die Bereitstellung von Windows-Containern in Ihrer lokalen Infrastruktur IaaS-VM

-   Ihre Organisation möglicherweise nicht in die Cloud verschieben können, oder es möglicherweise nicht für einen geschäftlichen Grund in die Cloud verschieben. Allerdings erhalten Sie dennoch die Vorteile der Verwendung von Windows-Containern in Ihren eigenen Rechenzentren.

-   Sie müssen möglicherweise andere Artefakte, die lokal eingesetzt werden, und Sie verlangsamt möglicherweise die die beim Versuch, in die Cloud verschieben. Z. B. Sicherheits- oder authentifizierungserweiterungen Abhängigkeiten mit einer lokalen Windows Server Active Directory oder einen anderen lokalen Bestand.

-   Wenn Sie noch heute mit der Windows-Container starten, können Sie eine Migration in Phasen in der Cloud morgen aus einer viel besseren Position vornehmen. Windows-Containern ist eine Bereitstellungseinheit für jeden Cloud, mit dem keine Sperre in gewinnt.
