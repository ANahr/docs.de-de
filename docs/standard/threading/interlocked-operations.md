# <a name="interlocked-operations"></a><span data-ttu-id="366af-102">Interlocked-Vorgänge</span><span class="sxs-lookup"><span data-stu-id="366af-102">Interlocked Operations</span></span>
<span data-ttu-id="366af-103">Die <xref:System.Threading.Interlocked>-Klasse stellt Methoden bereit, die den Zugriff auf eine Variable synchronisieren, die von mehreren Threads gemeinsam genutzt wird.</span><span class="sxs-lookup"><span data-stu-id="366af-103">The <xref:System.Threading.Interlocked> class provides methods that synchronize access to a variable that is shared by multiple threads.</span></span> <span data-ttu-id="366af-104">Threads aus verschiedenen Prozessen können diesen Mechanismus verwenden, wenn sich die Variable im freigegebenen Speicher befindet.</span><span class="sxs-lookup"><span data-stu-id="366af-104">The threads of different processes can use this mechanism if the variable is in shared memory.</span></span> <span data-ttu-id="366af-105">Interlocked-Vorgänge sind unteilbare (atomarische) Vorgänge. -   <span data-ttu-id="366af-109">In .NET Framework Version 2.0 liest die <xref:System.Threading.Interlocked.Read%2A>-Methode einen 64-Bit-Ganzzahlwert in einem atomaren Vorgang.</span><span class="sxs-lookup"><span data-stu-id="366af-109">In the .NET Framework version 2.0, the <xref:System.Threading.Interlocked.Read%2A> method reads a 64-bit integer value as an atomic operation.</span></span> <span data-ttu-id="366af-110">Dies ist von Vorteil für 32-Bit-Betriebssysteme, bei denen das Lesen eines 64-Bit-Ganzzahlwerts normalerweise keinen atomaren Vorgang darstellt.</span><span class="sxs-lookup"><span data-stu-id="366af-110">This is useful on 32-bit operating systems, where reading a 64-bit integer is not ordinarily an atomic operation.</span></span>

-   <span data-ttu-id="366af-112">Die <xref:System.Threading.Interlocked.Exchange%2A>-Methode führt einen atomaren Werteaustausch in einer bestimmten Variablen aus. -   <span data-ttu-id="366af-115">Die <xref:System.Threading.Interlocked.CompareExchange%2A>-Methode tauscht außerdem zwei Werte aus, jedoch abhängig vom Ergebnis eines Vergleichs.</span><span class="sxs-lookup"><span data-stu-id="366af-115">The <xref:System.Threading.Interlocked.CompareExchange%2A> method also exchanges two values, but contingent on the result of a comparison.</span></span> <span data-ttu-id="366af-116">In .NET Framework Version 2.0 kann eine generische Überladung dieser Methode verwendet werden, um diesen Austausch für eine Variable eines beliebigen Referenztyps auszuführen.</span><span class="sxs-lookup"><span data-stu-id="366af-116">In the .NET Framework version 2.0, a generic overload of this method can be used to perform this exchange on a variable of any reference type.</span></span> <span data-ttu-id="366af-117">Siehe <xref:System.Threading.Interlocked.CompareExchange%60%601%28%60%600%40%2C%60%600%2C%60%600%29>.</span><span class="sxs-lookup"><span data-stu-id="366af-117">See <xref:System.Threading.Interlocked.CompareExchange%60%601%28%60%600%40%2C%60%600%2C%60%600%29>.</span></span>  
  
 <span data-ttu-id="366af-118">Bei modernen Prozessoren können die Methoden der <xref:System.Threading.Interlocked>-Klasse oft durch eine einzige Anweisung implementiert werden.</span><span class="sxs-lookup"><span data-stu-id="366af-118">On modern processors, the methods of the <xref:System.Threading.Interlocked> class can often be implemented by a single instruction.</span></span> <span data-ttu-id="366af-119">Sie bieten folglich eine Hochleistungssynchronisierung und können verwendet werden, um Mechanismen für höherstufige Synchronisierung zu erstellen (z.B.
