# <a name="reader-writer-locks"></a><span data-ttu-id="8b49b-102">Lese-/Schreibsperren</span><span class="sxs-lookup"><span data-stu-id="8b49b-102">Reader-Writer Locks</span></span>
<span data-ttu-id="8b49b-103">Die <xref:System.Threading.ReaderWriterLockSlim>-Klasse ermöglicht mehreren Threads, eine Ressource gleichzeitig zu lesen, aber ein Thread muss zum Schreiben in die Ressource auf eine exklusive Sperre warten.</span><span class="sxs-lookup"><span data-stu-id="8b49b-103">The <xref:System.Threading.ReaderWriterLockSlim> class enables multiple threads to read a resource concurrently, but requires a thread to wait for an exclusive lock in order to write to the resource.</span></span>  
  
 <span data-ttu-id="8b49b-104">Sie könnten mit <xref:System.Threading.ReaderWriterLockSlim> in Ihrer Anwendung eine kooperative Synchronisierung zwischen Threads bereitstellen, die auf eine freigegebene Ressource zugreifen.</span><span class="sxs-lookup"><span data-stu-id="8b49b-104">You might use a <xref:System.Threading.ReaderWriterLockSlim> in your application to provide cooperative synchronization among threads that access a shared resource.</span></span> <span data-ttu-id="8b49b-105">Sperren werden auf <xref:System.Threading.ReaderWriterLockSlim> selbst vorgenommen.</span><span class="sxs-lookup"><span data-stu-id="8b49b-105">Locks are taken on the <xref:System.Threading.ReaderWriterLockSlim> itself.</span></span>  
  
 <span data-ttu-id="8b49b-106">Wie bei jedem Mechanismus zur Threadsynchronisierung müssen Sie sicherstellen, dass keine Threads die von <xref:System.Threading.ReaderWriterLockSlim> bereitgestellte Sperrung umgehen.</span><span class="sxs-lookup"><span data-stu-id="8b49b-106">As with any thread synchronization mechanism, you must ensure that no threads bypass the locking that is provided by <xref:System.Threading.ReaderWriterLockSlim>.</span></span> <span data-ttu-id="8b49b-107">Eine Möglichkeit, dies sicherzustellen, ist der Entwurf einer Klasse, die die freigegebene Ressource kapselt.</span><span class="sxs-lookup"><span data-stu-id="8b49b-107">One way to ensure this is to design a class that encapsulates the shared resource.</span></span> <span data-ttu-id="8b49b-108">Diese Klasse würde Member bereitstellen, die auf die private freigegebene Ressource zugreifen und ein privates <xref:System.Threading.ReaderWriterLockSlim> für die Synchronisierung nutzen.</span><span class="sxs-lookup"><span data-stu-id="8b49b-108">This class would provide members that access the private shared resource and that use a private <xref:System.Threading.ReaderWriterLockSlim> for synchronization.</span></span> <span data-ttu-id="8b49b-109">Ein Beispiel finden Sie im Codebeispiel für die <xref:System.Threading.ReaderWriterLockSlim>-Klasse.</span><span class="sxs-lookup"><span data-stu-id="8b49b-109">For an example, see the code example for the <xref:System.Threading.ReaderWriterLockSlim> class.</span></span> <span data-ttu-id="8b49b-110"><xref:System.Threading.ReaderWriterLockSlim> ist effizient genug zum Synchronisieren einzelner Objekte.</span><span class="sxs-lookup"><span data-stu-id="8b49b-110"><xref:System.Threading.ReaderWriterLockSlim> is efficient enough to be used to synchronize individual objects.</span></span>  
  
 <span data-ttu-id="8b49b-111">Strukturieren Sie Ihre Anwendung zum Minimieren der Dauer von Lese- und Schreibvorgängen.</span><span class="sxs-lookup"><span data-stu-id="8b49b-111">Structure your application to minimize the duration of read and write operations.</span></span> <span data-ttu-id="8b49b-112">Lange Schreibvorgänge beeinflussen den Durchsatz direkt, da die Schreibsperre exklusiv ist.</span><span class="sxs-lookup"><span data-stu-id="8b49b-112">Long write operations affect throughput directly because the write lock is exclusive.</span></span> <span data-ttu-id="8b49b-113">Lange Lesevorgänge blockieren wartende Schreiber, und mindestens ein Thread wartet auf den Schreibzugriff; Threads, die Lesezugriff anfordern, werden ebenfalls blockiert.</span><span class="sxs-lookup"><span data-stu-id="8b49b-113">Long read operations block waiting writers, and if at least one thread is waiting for write access, threads that request read access will be blocked as well.</span></span>
