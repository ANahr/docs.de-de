# Parallel Processing and Concurrency in the .NET Framework
.NET Framework bietet Ihnen verschiedene Möglichkeiten, mithilfe mehrerer Ausführungsthreads dafür zu sorgen, dass die Anwendung schnell auf Benutzeraktionen reagiert und die Leistung des Computers des Benutzers maximiert wird.  
  
## In diesem Abschnitt  
 [Threading](../../docs/standard/threading/index.md)  
 Beschreibt die grundlegenden Parallelitäts\- und Synchronisierungsmechanismen, die von .NET Framework bereitgestellt werden.  
  
 [Asynchronous Programming Patterns](../../docs/standard/asynchronous-programming-patterns/index.md)  
 Bietet eine kurze Übersicht über die drei Muster der asynchronen Programmierung, die in .NET Framework unterstützt werden:  
  
-   [Asynchronous Programming Model \(APM\)](../../docs/standard/asynchronous-programming-patterns/asynchronous-programming-model-apm.md) \(Legacy\)  
  
-   [Event\-based Asynchronous Pattern \(EAP\)](../../docs/standard/asynchronous-programming-patterns/event-based-asynchronous-pattern-eap.md) \(Legacy\)  
  
-   [Task\-based Asynchronous Pattern \(TAP\)](../../docs/standard/asynchronous-programming-patterns/task-based-asynchronous-pattern-tap.md) \(für die Neuentwicklung empfohlen\)  
  
 [Parallel Programming](../../docs/standard/parallel-programming/index.md)  
 Beschreibt ein aufgabenbasiertes Programmiermodell, das die parallele Entwicklung vereinfacht, sodass Sie effizienten, differenzierten und skalierbaren parallelen Code in einer natürlichen Sprache schreiben können, ohne direkt mit Threads oder dem Threadpool arbeiten zu müssen.
