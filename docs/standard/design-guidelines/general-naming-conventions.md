## <a name="word-choice"></a><span data-ttu-id="47bd2-104">Word-Auswahl</span><span class="sxs-lookup"><span data-stu-id="47bd2-104">Word Choice</span></span>  
 <span data-ttu-id="47bd2-105">**Führen Sie ✓** übersichtlich Bezeichnernamen auswählen.</span><span class="sxs-lookup"><span data-stu-id="47bd2-105">**✓ DO** choose easily readable identifier names.</span></span>  
  
 <span data-ttu-id="47bd2-106">Angenommen, eine Eigenschaft namens `HorizontalAlignment` ist Englisch-lesbarer als `AlignmentHorizontal`.</span><span class="sxs-lookup"><span data-stu-id="47bd2-106">For example, a property named `HorizontalAlignment` is more English-readable than `AlignmentHorizontal`.</span></span>  
  
 <span data-ttu-id="47bd2-107">**Führen Sie ✓** ziehen Sie Lesbarkeit Knappheit zu finden.</span><span class="sxs-lookup"><span data-stu-id="47bd2-107">**✓ DO** favor readability over brevity.</span></span>  
  
 <span data-ttu-id="47bd2-108">Der Eigenschaftenname `CanScrollHorizontally` ist besser als `ScrollableX` (eine unbekannte Verweis auf die X-Achse).</span><span class="sxs-lookup"><span data-stu-id="47bd2-108">The property name `CanScrollHorizontally` is better than `ScrollableX` (an obscure reference to the X-axis).</span></span>  
  
 <span data-ttu-id="47bd2-109">**X nicht** Unterstriche, Bindestriche oder andere nicht alphanumerischen Zeichen verwenden.</span><span class="sxs-lookup"><span data-stu-id="47bd2-109">**X DO NOT** use underscores, hyphens, or any other nonalphanumeric characters.</span></span>  
  
 <span data-ttu-id="47bd2-110">**X nicht** verwenden Ungarischer Notation benannt.</span><span class="sxs-lookup"><span data-stu-id="47bd2-110">**X DO NOT** use Hungarian notation.</span></span>  
  
 <span data-ttu-id="47bd2-111">**X vermeiden** Bezeichner, die häufig mit Schlüsselwörtern in Konflikt mit der übrigen Programmiersprachen verwendet.</span><span class="sxs-lookup"><span data-stu-id="47bd2-111">**X AVOID** using identifiers that conflict with keywords of widely used programming languages.</span></span>  
  
 <span data-ttu-id="47bd2-112">Gemäß der Regel 4 der Common Language Specification (CLS) müssen alle kompatible Sprachen einen Mechanismus bereit, der Zugriff auf benannte Elemente ermöglicht, die ein Schlüsselwort für diese Sprache nicht als Bezeichner verwendet.</span><span class="sxs-lookup"><span data-stu-id="47bd2-112">According to Rule 4 of the Common Language Specification (CLS), all compliant languages must provide a mechanism that allows access to named items that use a keyword of that language as an identifier.</span></span> <span data-ttu-id="47bd2-113">C#-, verwendet z. B. das @-Zeichen als Escapezeichen Mechanismus in diesem Fall.</span><span class="sxs-lookup"><span data-stu-id="47bd2-113">C#, for example, uses the @ sign as an escape mechanism in this case.</span></span> <span data-ttu-id="47bd2-114">Allerdings ist es immer noch eine gute Idee, häufige oder gemeinsame Schlüsselwörter vermeiden, da es sehr viel schwieriger, eine Methode mit der-Escapesequenz als ohne sie zu verwenden ist.</span><span class="sxs-lookup"><span data-stu-id="47bd2-114">However, it is still a good idea to avoid common keywords because it is much more difficult to use a method with the escape sequence than one without it.</span></span>  
  
## <a name="using-abbreviations-and-acronyms"></a><span data-ttu-id="47bd2-115">Verwenden von Abkürzungen und Akronyme</span><span class="sxs-lookup"><span data-stu-id="47bd2-115">Using Abbreviations and Acronyms</span></span>  
 <span data-ttu-id="47bd2-116">**X nicht** Abkürzungen oder Kontraktionen als Teil des Bezeichnernamen verwenden.</span><span class="sxs-lookup"><span data-stu-id="47bd2-116">**X DO NOT** use abbreviations or contractions as part of identifier names.</span></span>  
  
 <span data-ttu-id="47bd2-117">Verwenden Sie z. B. `GetWindow` statt `GetWin`.</span><span class="sxs-lookup"><span data-stu-id="47bd2-117">For example, use `GetWindow` rather than `GetWin`.</span></span>  
  
 <span data-ttu-id="47bd2-118">**X nicht** verwenden Sie Akronyme, die nicht weit verbreitete und sogar, wenn sie, falls erforderlich sind.</span><span class="sxs-lookup"><span data-stu-id="47bd2-118">**X DO NOT** use any acronyms that are not widely accepted, and even if they are, only when necessary.</span></span>

 <span data-ttu-id="47bd2-121">Beispielsweise `GetLength` ist ein besserer Name als `GetInt`.</span><span class="sxs-lookup"><span data-stu-id="47bd2-121">For example, `GetLength` is a better name than `GetInt`.</span></span>  
  
 <span data-ttu-id="47bd2-122">**Führen Sie ✓** verwenden Sie eine generische CLR-Typnamen, anstatt einen sprachspezifischen Namen, in den seltenen Fällen, wenn ein Bezeichner keine semantische Bedeutung außer seinem Typ aufweist.</span><span class="sxs-lookup"><span data-stu-id="47bd2-122">**✓ DO** use a generic CLR type name, rather than a language-specific name, in the rare cases when an identifier has no semantic meaning beyond its type.</span></span>

 <span data-ttu-id="47bd2-188">Dadurch wird um die Beziehung zwischen der APIs zu markieren.</span><span class="sxs-lookup"><span data-stu-id="47bd2-188">This helps to highlight the relationship between the APIs.</span></span>  
  
 <span data-ttu-id="47bd2-189">**Führen Sie ✓** bevorzugen Sie ein Suffix anstelle ein Präfix an, dass eine neue Version einer vorhandenen API hinzufügen.</span><span class="sxs-lookup"><span data-stu-id="47bd2-189">**✓ DO** prefer adding a suffix rather than a prefix to indicate a new version of an existing API.</span></span>  
  
 <span data-ttu-id="47bd2-190">Dadurch wird die Ermittlung unterstützt, beim Durchsuchen der Dokumentation, oder Verwenden von IntelliSense.</span><span class="sxs-lookup"><span data-stu-id="47bd2-190">This will assist discovery when browsing documentation, or using IntelliSense.</span></span> <span data-ttu-id="47bd2-191">Die alte Version der API wird in der Nähe der neuen APIs organisiert werden, da die meisten Browsern und IntelliSense Bezeichner in alphabetischer Reihenfolge anzuzeigen.</span><span class="sxs-lookup"><span data-stu-id="47bd2-191">The old version of the API will be organized close to the new APIs, because most browsers and IntelliSense show identifiers in alphabetical order.</span></span>

 <span data-ttu-id="47bd2-193">**Führen Sie ✓** ein numerisches Suffix verwenden, um eine neue Version einer vorhandenen API anzugeben, insbesondere, wenn der vorhandene Name der API der einzige Name, die sinnvoll ist (d. h., wenn es sich um einen Industriestandard ist) und wenn jeder sinnvolle hinzufügen Suffix (oder Ändern des Namens) keine app Ropriate-Option.</span><span class="sxs-lookup"><span data-stu-id="47bd2-193">**✓ DO** use a numeric suffix to indicate a new version of an existing API, particularly if the existing name of the API is the only name that makes sense (i.e., if it is an industry standard) and if adding any meaningful suffix (or changing the name) is not an appropriate option.</span></span>

Führen Sie es für die neuen APIs mit nur einer 64-Bit-Version.</span><span class="sxs-lookup"><span data-stu-id="47bd2-196">You only need to take this approach when the existing 32-bit API exists; don’t do it for brand new APIs with only a 64-bit version.</span></span>
