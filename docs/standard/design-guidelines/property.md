# <a name="property-design"></a><span data-ttu-id="a1270-102">Eigenschaftenentwurf</span><span class="sxs-lookup"><span data-stu-id="a1270-102">Property Design</span></span>
<span data-ttu-id="a1270-103">Obwohl die Eigenschaften, Methoden technisch sehr ähnlich sind, sind sie sehr unterschiedliche im Hinblick auf ihre Szenarien für die Verwendung.</span><span class="sxs-lookup"><span data-stu-id="a1270-103">Although properties are technically very similar to methods, they are quite different in terms of their usage scenarios.</span></span> <span data-ttu-id="a1270-104">Sie sollte als intelligente Felder angezeigt werden.</span><span class="sxs-lookup"><span data-stu-id="a1270-104">They should be seen as smart fields.</span></span> <span data-ttu-id="a1270-105">Sie verfügen über die Funktionsaufrufsyntax von Feldern und die Flexibilität der Methoden.</span><span class="sxs-lookup"><span data-stu-id="a1270-105">They have the calling syntax of fields, and the flexibility of methods.</span></span>  
  
 <span data-ttu-id="a1270-106">**Führen Sie ✓** schreibgeschützte Eigenschaften erstellen, wenn der Aufrufer nicht der Wert der Eigenschaft geändert werden soll.</span><span class="sxs-lookup"><span data-stu-id="a1270-106">**✓ DO** create get-only properties if the caller should not be able to change the value of the property.</span></span>  
  
 <span data-ttu-id="a1270-107">Bedenken Sie, dass bei den Typ der Eigenschaft einen änderbaren Referenztyp ist, den Wert der Eigenschaft kann geändert werden, auch wenn die Eigenschaft nur Get-ist.</span><span class="sxs-lookup"><span data-stu-id="a1270-107">Keep in mind that if the type of the property is a mutable reference type, the property value can be changed even if the property is get-only.</span></span>  
  
 <span data-ttu-id="a1270-108">**X nicht** Bereitstellen mit der Setter müssen umfassenderen Zugriff hat als der Getter für eine lesegeschützte Eigenschaften oder Eigenschaften.</span><span class="sxs-lookup"><span data-stu-id="a1270-108">**X DO NOT** provide set-only properties or properties with the setter having broader accessibility than the getter.</span></span>  
  
 <span data-ttu-id="a1270-109">Verwenden Sie z. B. Eigenschaften nicht mit einem öffentlichen Setter und einen geschützten Getter.</span><span class="sxs-lookup"><span data-stu-id="a1270-109">For example, do not use properties with a public setter and a protected getter.</span></span>  
  
 <span data-ttu-id="a1270-110">Wenn der Getter für eine Eigenschaft kann nicht bereitgestellt werden, implementieren Sie stattdessen die Funktionen wie eine Methode.</span><span class="sxs-lookup"><span data-stu-id="a1270-110">If the property getter cannot be provided, implement the functionality as a method instead.</span></span> <span data-ttu-id="a1270-111">Beachten Sie, beginnend mit den Methodennamen `Set` und führen Sie mit, was Sie die Eigenschaft mit dem Namen verfügen würde.</span><span class="sxs-lookup"><span data-stu-id="a1270-111">Consider starting the method name with `Set` and follow with what you would have named the property.</span></span> <span data-ttu-id="a1270-112">Beispielsweise <xref:System.AppDomain> verfügt über eine Methode namens `SetCachePath` anstatt eine lesegeschützte Eigenschaft mit dem Namen `CachePath`.</span><span class="sxs-lookup"><span data-stu-id="a1270-112">For example, <xref:System.AppDomain> has a method called `SetCachePath` instead of having a set-only property called `CachePath`.</span></span>  
  
 <span data-ttu-id="a1270-113">**Führen Sie ✓** sensibler Standardwerte für alle Eigenschaften, um sicherzustellen, dass die Standardwerte nicht in einer Sicherheitslücke oder sehr ineffizienten Code führen zur Verfügung gestellt.</span><span class="sxs-lookup"><span data-stu-id="a1270-113">**✓ DO** provide sensible default values for all properties, ensuring that the defaults do not result in a security hole or terribly inefficient code.</span></span>  
  
 <span data-ttu-id="a1270-114">**Führen Sie ✓** können Eigenschaften in beliebiger Reihenfolge festgelegt werden, auch wenn dadurch ein temporärer ungültigen Zustand des Objekts.</span><span class="sxs-lookup"><span data-stu-id="a1270-114">**✓ DO** allow properties to be set in any order even if this results in a temporary invalid state of the object.</span></span>  
  
 <span data-ttu-id="a1270-115">Es ist üblich für zwei oder mehr Eigenschaften zu einem Zeitpunkt zusammenhängende sein, in denen einige Werte einer Eigenschaft möglicherweise ungültig, die Werte der anderen Eigenschaften für das gleiche Objekt angegeben.</span><span class="sxs-lookup"><span data-stu-id="a1270-115">It is common for two or more properties to be interrelated to a point where some values of one property might be invalid given the values of other properties on the same object.</span></span> <span data-ttu-id="a1270-116">In solchen Fällen sollten aufgrund von ungültigen Status ausgelösten Ausnahmen verschoben werden, bis die zusammenhängenden Eigenschaften zusammen tatsächlich vom Objekt verwendet werden.</span><span class="sxs-lookup"><span data-stu-id="a1270-116">In such cases, exceptions resulting from the invalid state should be postponed until the interrelated properties are actually used together by the object.</span></span>  
  
 <span data-ttu-id="a1270-117">**Führen Sie ✓** den vorherigen Wert beibehalten, wenn ein Eigenschaftensetter eine Ausnahme auslöst.</span><span class="sxs-lookup"><span data-stu-id="a1270-117">**✓ DO** preserve the previous value if a property setter throws an exception.</span></span>  
  
 <span data-ttu-id="a1270-118">**X vermeiden** Auslösen von Ausnahmen von den Eigenschaftengetter.</span><span class="sxs-lookup"><span data-stu-id="a1270-118">**X AVOID** throwing exceptions from property getters.</span></span>  
  
 <span data-ttu-id="a1270-119">Eigenschaftengetter darf sollten einfache Operationen sein und keine Vorbedingungen.</span><span class="sxs-lookup"><span data-stu-id="a1270-119">Property getters should be simple operations and should not have any preconditions.</span></span> <span data-ttu-id="a1270-120">Wenn eine getter-Methode eine Ausnahme auslösen kann, sollte es eine Methode wahrscheinlich überarbeitet.</span><span class="sxs-lookup"><span data-stu-id="a1270-120">If a getter can throw an exception, it should probably be redesigned to be a method.</span></span> <span data-ttu-id="a1270-121">Beachten Sie, dass diese Regel nicht für Indexer ist gilt, in dem wir Ausnahmen als Ergebnis überprüft die Argumente erwarten.</span><span class="sxs-lookup"><span data-stu-id="a1270-121">Notice that this rule does not apply to indexers, where we do expect exceptions as a result of validating the arguments.</span></span>  
  
### <a name="indexed-property-design"></a><span data-ttu-id="a1270-122">Indizierte Eigenschaftenentwurf</span><span class="sxs-lookup"><span data-stu-id="a1270-122">Indexed Property Design</span></span>  
 <span data-ttu-id="a1270-123">Eine indizierte Eigenschaft ist eine spezielle Eigenschaft, die Parameter und kann mit speziellen Syntax ähnelt derjenigen Arrayindizierung aufgerufen werden.</span><span class="sxs-lookup"><span data-stu-id="a1270-123">An indexed property is a special property that can have parameters and can be called with special syntax similar to array indexing.</span></span>  
  
 <span data-ttu-id="a1270-124">Indizierte Eigenschaften werden häufig als Indexer bezeichnet.</span><span class="sxs-lookup"><span data-stu-id="a1270-124">Indexed properties are commonly referred to as indexers.</span></span> <span data-ttu-id="a1270-125">Indexer sollte nur in-APIs verwendet werden, die Zugriff auf Elemente in einer logischen Auflistung bereitstellen.</span><span class="sxs-lookup"><span data-stu-id="a1270-125">Indexers should be used only in APIs that provide access to items in a logical collection.</span></span> <span data-ttu-id="a1270-126">Eine Zeichenfolge ist z. B. eine Auflistung von Zeichen oder der Indexer auf <xref:System.String?displayProperty=nameWithType> wurde hinzugefügt, um die Zeichen zugreifen.</span><span class="sxs-lookup"><span data-stu-id="a1270-126">For example, a string is a collection of characters, and the indexer on <xref:System.String?displayProperty=nameWithType> was added to access its characters.</span></span>  
  
 <span data-ttu-id="a1270-127">**✓ GGF.** Verwenden von Indexern für den Zugriff auf Daten in einem internen Array gespeichert.</span><span class="sxs-lookup"><span data-stu-id="a1270-127">**✓ CONSIDER** using indexers to provide access to data stored in an internal array.</span></span>  
  
 <span data-ttu-id="a1270-128">**✓ GGF.** bereitstellen Indexer auf Typen, die Auflistungen von Elementen darstellt.</span><span class="sxs-lookup"><span data-stu-id="a1270-128">**✓ CONSIDER** providing indexers on types representing collections of items.</span></span>  
  
 <span data-ttu-id="a1270-129">**X vermeiden** mit indizierten Eigenschaften mit mehreren Parametern.</span><span class="sxs-lookup"><span data-stu-id="a1270-129">**X AVOID** using indexed properties with more than one parameter.</span></span>  
  
 <span data-ttu-id="a1270-130">Wenn der Entwurf mehrere Parameter erfordert, überdenken Sie, ob die Eigenschaft wirklich einen Accessor einer logischen Auflistung darstellt.</span><span class="sxs-lookup"><span data-stu-id="a1270-130">If the design requires multiple parameters, reconsider whether the property really represents an accessor to a logical collection.</span></span> <span data-ttu-id="a1270-131">Wenn dies nicht der Fall ist, verwenden Sie stattdessen die Methoden.</span><span class="sxs-lookup"><span data-stu-id="a1270-131">If it does not, use methods instead.</span></span> <span data-ttu-id="a1270-132">Beachten Sie, beginnend mit den Methodennamen `Get` oder `Set`.</span><span class="sxs-lookup"><span data-stu-id="a1270-132">Consider starting the method name with `Get` or `Set`.</span></span>  
  
 <span data-ttu-id="a1270-133">**X vermeiden** Indexer mit Parametertypen außer <xref:System.Int32?displayProperty=nameWithType>, <xref:System.Int64?displayProperty=nameWithType>, <xref:System.String?displayProperty=nameWithType>, <xref:System.Object?displayProperty=nameWithType>, oder eine Enumeration.</span><span class="sxs-lookup"><span data-stu-id="a1270-133">**X AVOID** indexers with parameter types other than <xref:System.Int32?displayProperty=nameWithType>, <xref:System.Int64?displayProperty=nameWithType>, <xref:System.String?displayProperty=nameWithType>, <xref:System.Object?displayProperty=nameWithType>, or an enum.</span></span>  
  
 <span data-ttu-id="a1270-134">Wenn der Entwurf andere Parametertypen erfordert, auswerten Sie stark neu, ob die API tatsächlich einen Accessor einer logischen Auflistung darstellt.</span><span class="sxs-lookup"><span data-stu-id="a1270-134">If the design requires other types of parameters, strongly reevaluate whether the API really represents an accessor to a logical collection.</span></span> <span data-ttu-id="a1270-135">Wenn dies nicht der Fall ist, verwenden Sie eine Methode an.</span><span class="sxs-lookup"><span data-stu-id="a1270-135">If it does not, use a method.</span></span> <span data-ttu-id="a1270-136">Beachten Sie, beginnend mit den Methodennamen `Get` oder `Set`.</span><span class="sxs-lookup"><span data-stu-id="a1270-136">Consider starting the method name with `Get` or `Set`.</span></span>  
  
 <span data-ttu-id="a1270-137">**Führen Sie ✓** verwenden Sie den Namen `Item` für indizierte Eigenschaften, es sei denn, es ein deutlich besserer Name ist (z. B. finden Sie unter der <xref:System.String.Chars%2A> Eigenschaft auf `System.String`).</span><span class="sxs-lookup"><span data-stu-id="a1270-137">**✓ DO** use the name `Item` for indexed properties unless there is an obviously better name (e.g., see the <xref:System.String.Chars%2A> property on `System.String`).</span></span>  
  
 <span data-ttu-id="a1270-138">Sind Indexer in c# ist standardmäßig mit dem Namen Element.</span><span class="sxs-lookup"><span data-stu-id="a1270-138">In C#, indexers are by default named Item.</span></span> <span data-ttu-id="a1270-139">Die <xref:System.Runtime.CompilerServices.IndexerNameAttribute> zum Anpassen dieses Namens verwendet werden können.</span><span class="sxs-lookup"><span data-stu-id="a1270-139">The <xref:System.Runtime.CompilerServices.IndexerNameAttribute> can be used to customize this name.</span></span>  
  
 <span data-ttu-id="a1270-140">**X nicht** Geben Sie einen Indexer und Methoden, die semantisch gleichwertig sind.</span><span class="sxs-lookup"><span data-stu-id="a1270-140">**X DO NOT** provide both an indexer and methods that are semantically equivalent.</span></span>  
  
 <span data-ttu-id="a1270-141">**X nicht** bieten mehr als eine Familie von überladene Indexer in einem Typ.</span><span class="sxs-lookup"><span data-stu-id="a1270-141">**X DO NOT** provide more than one family of overloaded indexers in one type.</span></span>  
  
 <span data-ttu-id="a1270-142">Dies wird vom C#-Compiler erzwungen.</span><span class="sxs-lookup"><span data-stu-id="a1270-142">This is enforced by the C# compiler.</span></span>  
  
 <span data-ttu-id="a1270-143">**X nicht** verwenden nicht standardmäßiger indizierte Eigenschaften.</span><span class="sxs-lookup"><span data-stu-id="a1270-143">**X DO NOT** use nondefault indexed properties.</span></span>  
  
 <span data-ttu-id="a1270-144">Dies wird vom C#-Compiler erzwungen.</span><span class="sxs-lookup"><span data-stu-id="a1270-144">This is enforced by the C# compiler.</span></span>  
  
### <a name="property-change-notification-events"></a><span data-ttu-id="a1270-145">Eigenschaftenänderungsereignisse-Benachrichtigung</span><span class="sxs-lookup"><span data-stu-id="a1270-145">Property Change Notification Events</span></span>  
 <span data-ttu-id="a1270-146">Manchmal ist es nützlich, um ein Ereignis informiert den Benutzer über Änderungen eines Eigenschaftswerts bereitstellen.</span><span class="sxs-lookup"><span data-stu-id="a1270-146">Sometimes it is useful to provide an event notifying the user of changes in a property value.</span></span> <span data-ttu-id="a1270-147">Beispielsweise `System.Windows.Forms.Control` löst eine `TextChanged` Ereignis nach dem Wert von dessen `Text` -Eigenschaft geändert hat.</span><span class="sxs-lookup"><span data-stu-id="a1270-147">For example, `System.Windows.Forms.Control` raises a `TextChanged` event after the value of its `Text` property has changed.</span></span>  
  
 <span data-ttu-id="a1270-148">**✓ GGF.** auslösen ändern Benachrichtigungsereignisse Wenn Eigenschaftswerte in allgemeine APIs (in der Regel Designer-Komponenten) geändert werden.</span><span class="sxs-lookup"><span data-stu-id="a1270-148">**✓ CONSIDER** raising change notification events when property values in high-level APIs (usually designer components) are modified.</span></span>  
  
 <span data-ttu-id="a1270-149">Ist ein gutes Szenario für Benutzer erkennt, wenn eine Eigenschaft eines Objekts geändert wird, sollte das Objekt eine änderungsbenachrichtigung für die Eigenschaft auslösen.</span><span class="sxs-lookup"><span data-stu-id="a1270-149">If there is a good scenario for a user to know when a property of an object is changing, the object should raise a change notification event for the property.</span></span>  
  
 <span data-ttu-id="a1270-150">Allerdings ist es unwahrscheinlich, lohnt des Mehraufwands für solche Ereignisse auslösen, für die Low-Level-APIs wie z. B. Basistypen oder Auflistungen sein.</span><span class="sxs-lookup"><span data-stu-id="a1270-150">However, it is unlikely to be worth the overhead to raise such events for low-level APIs such as base types or collections.</span></span> <span data-ttu-id="a1270-151">Beispielsweise <xref:System.Collections.Generic.List%601> solche Ereignisse aus, wenn der Liste ein neues Element hinzugefügt wird nicht ausgelöst werden soll und die `Count` -Eigenschaft ändert.</span><span class="sxs-lookup"><span data-stu-id="a1270-151">For example, <xref:System.Collections.Generic.List%601> would not raise such events when a new item is added to the list and the `Count` property changes.</span></span>  
  
 <span data-ttu-id="a1270-152">**✓ GGF.** Auslösen von Benachrichtigungsereignissen, wenn der Wert einer Eigenschaft über externe erzwingt ändert ändern.</span><span class="sxs-lookup"><span data-stu-id="a1270-152">**✓ CONSIDER** raising change notification events when the value of a property changes via external forces.</span></span>  
  
 <span data-ttu-id="a1270-153">Wenn Sie ein Eigenschaftswert über einige externe Force (in einer Weise als durch Aufrufen von Methoden für das Objekt) geändert wird, löst Ereignisse für den Entwickler anzugeben, dass der Wert ändert sich und geändert hat.</span><span class="sxs-lookup"><span data-stu-id="a1270-153">If a property value changes via some external force (in a way other than by calling methods on the object), raise events indicate to the developer that the value is changing and has changed.</span></span> <span data-ttu-id="a1270-154">Ein gutes Beispiel ist die `Text` Eigenschaft für ein Textfeld-Steuerelement.</span><span class="sxs-lookup"><span data-stu-id="a1270-154">A good example is the `Text` property of a text box control.</span></span> <span data-ttu-id="a1270-155">Wenn der Benutzer eingibt Text in einem `TextBox`, der Eigenschaftswert automatisch geändert.</span><span class="sxs-lookup"><span data-stu-id="a1270-155">When the user types text in a `TextBox`, the property value automatically changes.</span></span>
