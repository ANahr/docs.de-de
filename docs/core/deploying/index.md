## <a name="framework-dependent-deployments-fdd"></a><span data-ttu-id="d1aaf-115">Framework-abhängige Bereitstellungen (FDD)</span><span class="sxs-lookup"><span data-stu-id="d1aaf-115">Framework-dependent deployments (FDD)</span></span>

### <a name="why-create-a-framework-dependent-deployment"></a><span data-ttu-id="d1aaf-119">Warum eine Framework-abhängige Bereitstellung erstellen?</span><span class="sxs-lookup"><span data-stu-id="d1aaf-119">Why create a framework-dependent deployment?</span></span>

- <span data-ttu-id="d1aaf-124">Ihr Bereitstellungspaket ist klein.</span><span class="sxs-lookup"><span data-stu-id="d1aaf-124">The size of your deployment package is small.</span></span> <span data-ttu-id="d1aaf-125">Sie müssen nur Ihre Anwendung und deren Abhängigkeiten bereitstellen, nicht .NET Core selbst.</span><span class="sxs-lookup"><span data-stu-id="d1aaf-125">You only deploy your app and its dependencies, not .NET Core itself.</span></span>

- <span data-ttu-id="d1aaf-126">Mehrere Anwendungen verwenden die gleiche .NET Core-Installation, die Speicherplatz und Arbeitsspeicher auf dem Hostsystem verringert.</span><span class="sxs-lookup"><span data-stu-id="d1aaf-126">Multiple apps use the same .NET Core installation, which reduces both disk space and memory usage on host systems.</span></span>

<span data-ttu-id="d1aaf-127">Es gibt auch einige Nachteile:</span><span class="sxs-lookup"><span data-stu-id="d1aaf-127">There are also a few disadvantages:</span></span>

- <span data-ttu-id="d1aaf-129">Es ist möglich, dass die .NET Core Runtime und die Bibliotheken ohne Ihr Wissen in zukünftigen Versionen geändert werden.</span><span class="sxs-lookup"><span data-stu-id="d1aaf-129">It's possible for the .NET Core runtime and libraries to change without your knowledge in future releases.</span></span> <span data-ttu-id="d1aaf-130">In seltenen Fällen kann dies das Verhalten Ihrer Anwendung ändern.</span><span class="sxs-lookup"><span data-stu-id="d1aaf-130">In rare cases, this may change the behavior of your app.</span></span>

## <a name="self-contained-deployments-scd"></a><span data-ttu-id="d1aaf-131">Eigenständige Bereitstellungen (Self-contained deployments, SCD)</span><span class="sxs-lookup"><span data-stu-id="d1aaf-131">Self-contained deployments (SCD)</span></span>

<span data-ttu-id="d1aaf-132">Bei einer eigenständigen Bereitstellung stellen Sie nicht nur Ihre Anwendung und alle erforderlichen Drittanbieterabhängigkeiten bereit, sondern auch die .NET Core-Version, mit der Sie Ihre Anwendung erstellt haben.</span><span class="sxs-lookup"><span data-stu-id="d1aaf-132">For a self-contained deployment, you deploy your app and any required third-party dependencies along with the version of .NET Core that you used to build the app.</span></span> <span data-ttu-id="d1aaf-133">Eine eigenständige Bereitstellung schließt allerdings nicht die [nativen Abhängigkeiten von .NET Core](https://github.com/dotnet/core/blob/master/Documentation/prereqs.md) auf verschiedenen Plattformen mit ein. Diese müssen daher vor dem Ausführen der Anwendung installiert werden.</span><span class="sxs-lookup"><span data-stu-id="d1aaf-133">Creating an SCD doesn't include the [native dependencies of .NET Core](https://github.com/dotnet/core/blob/master/Documentation/prereqs.md) on various platforms, so these must be present before the app runs.</span></span>

<span data-ttu-id="d1aaf-134">Framework-abhängige Bereitstellungen (FDD) und eigenständige Bereitstellungen (SCD) verwenden getrennte ausführbare Hostdateien, sodass Sie eine ausführbare Hostdatei für eine SCD mit Ihrer Herausgebersignatur signieren können.</span><span class="sxs-lookup"><span data-stu-id="d1aaf-134">FDD and SCD deployments use separate host executables, so you can sign a host executable for an SCD with your publisher signature.</span></span>

<span data-ttu-id="d1aaf-136">Das Bereitstellen einer eigenständigen Bereitstellung hat zwei wesentliche Vorteile:</span><span class="sxs-lookup"><span data-stu-id="d1aaf-136">Deploying a Self-contained deployment has two major advantages:</span></span>

- <span data-ttu-id="d1aaf-139">Sie können sicher sein, dass das Zielsystem Ihre .NET Core-Anwendung ausführen kann, da Sie die .NET Core-Version bereitstellen, die darauf ausgeführt werden soll.</span><span class="sxs-lookup"><span data-stu-id="d1aaf-139">You can be assured that the target system can run your .NET Core app, since you're providing the version of .NET Core that it will run on.</span></span>

<span data-ttu-id="d1aaf-140">Es hat auch einige Nachteile:</span><span class="sxs-lookup"><span data-stu-id="d1aaf-140">It also has a number of disadvantages:</span></span>

- <span data-ttu-id="d1aaf-141">Da .NET Core in Ihrem Bereitstellungspaket enthalten ist, müssen Sie die Zielplattformen auswählen, für die Sie im Voraus Bereitstellungspaketen erstellen.</span><span class="sxs-lookup"><span data-stu-id="d1aaf-141">Because .NET Core is included in your deployment package, you must select the target platforms for which you build deployment packages in advance.</span></span>

- <span data-ttu-id="d1aaf-142">Die Größe Ihres Bereitstellungspakets ist relativ groß, da es auch .NET Core, Ihre Anwendung und ihre Drittanbieter-Abhängigkeiten enthält.</span><span class="sxs-lookup"><span data-stu-id="d1aaf-142">The size of your deployment package is relatively large, since you have to include .NET Core as well as your app and its third-party dependencies.</span></span>

- <span data-ttu-id="d1aaf-143">Das Bereitstellen von zahlreichen eigenständigen .NET Core-Anwendungen auf ein System kann viel Speicherplatz verbrauchen, da jede Anwendung .NET Core-Dateien dupliziert.</span><span class="sxs-lookup"><span data-stu-id="d1aaf-143">Deploying numerous self-contained .NET Core apps to a system can consume significant amounts of disk space, since each app duplicates .NET Core files.</span></span>
