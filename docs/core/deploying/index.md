## <a name="framework-dependent-deployments-fdd"></a><span data-ttu-id="c325e-115">Framework-abhängige Bereitstellungen (FDD)</span><span class="sxs-lookup"><span data-stu-id="c325e-115">Framework-dependent deployments (FDD)</span></span>

### <a name="why-create-a-framework-dependent-deployment"></a><span data-ttu-id="c325e-119">Warum eine Framework-abhängige Bereitstellung erstellen?</span><span class="sxs-lookup"><span data-stu-id="c325e-119">Why create a framework-dependent deployment?</span></span>

<span data-ttu-id="c325e-120">Die Bereitstellung einer FDD hat eine Reihe von Vorteilen:</span><span class="sxs-lookup"><span data-stu-id="c325e-120">Deploying an FDD has a number of advantages:</span></span>

- <span data-ttu-id="c325e-124">Ihr Bereitstellungspaket ist klein.</span><span class="sxs-lookup"><span data-stu-id="c325e-124">The size of your deployment package is small.</span></span> <span data-ttu-id="c325e-125">Sie müssen nur Ihre Anwendung und deren Abhängigkeiten bereitstellen, nicht .NET Core selbst.</span><span class="sxs-lookup"><span data-stu-id="c325e-125">You only deploy your app and its dependencies, not .NET Core itself.</span></span>

- <span data-ttu-id="c325e-126">Mehrere Anwendungen verwenden die gleiche .NET Core-Installation, die Speicherplatz und Arbeitsspeicher auf dem Hostsystem verringert.</span><span class="sxs-lookup"><span data-stu-id="c325e-126">Multiple apps use the same .NET Core installation, which reduces both disk space and memory usage on host systems.</span></span>

<span data-ttu-id="c325e-127">Es gibt auch einige Nachteile:</span><span class="sxs-lookup"><span data-stu-id="c325e-127">There are also a few disadvantages:</span></span>

## <a name="self-contained-deployments-scd"></a><span data-ttu-id="c325e-131">Eigenständige Bereitstellungen (Self-contained deployments, SCD)</span><span class="sxs-lookup"><span data-stu-id="c325e-131">Self-contained deployments (SCD)</span></span>

<span data-ttu-id="c325e-132">Bei einer eigenständigen Bereitstellung stellen Sie nicht nur Ihre Anwendung und alle erforderlichen Drittanbieterabhängigkeiten bereit, sondern auch die .NET Core-Version, mit der Sie Ihre Anwendung erstellt haben.</span><span class="sxs-lookup"><span data-stu-id="c325e-132">For a self-contained deployment, you deploy your app and any required third-party dependencies along with the version of .NET Core that you used to build the app.</span></span> <span data-ttu-id="c325e-133">Eine eigenständige Bereitstellung schließt allerdings nicht die [nativen Abhängigkeiten von .NET Core](https://github.com/dotnet/core/blob/master/Documentation/prereqs.md) auf verschiedenen Plattformen mit ein. Diese müssen daher vor dem Ausführen der Anwendung installiert werden.</span><span class="sxs-lookup"><span data-stu-id="c325e-133">Creating an SCD doesn't include the [native dependencies of .NET Core](https://github.com/dotnet/core/blob/master/Documentation/prereqs.md) on various platforms, so these must be present before the app runs.</span></span> <span data-ttu-id="c325e-134">Weitere Informationen zur Versionsbindung zur Laufzeit finden Sie im Artikel zur [Versionsbindung in .NET Core](../versions/selection.md).</span><span class="sxs-lookup"><span data-stu-id="c325e-134">For more information on version binding at runtime, see the article on [version binding in .NET Core](../versions/selection.md).</span></span>

The serviced runtime does not have to be present on your build system; it is downloaded automatically from NuGet.org. <span data-ttu-id="c325e-138">Framework-abhängige Bereitstellungen (FDD) und eigenständige Bereitstellungen (SCD) verwenden getrennte ausführbare Hostdateien, sodass Sie eine ausführbare Hostdatei für eine SCD mit Ihrer Herausgebersignatur signieren können.</span><span class="sxs-lookup"><span data-stu-id="c325e-138">FDD and SCD deployments use separate host executables, so you can sign a host executable for an SCD with your publisher signature.</span></span>

<span data-ttu-id="c325e-140">Das Bereitstellen einer eigenständigen Bereitstellung hat zwei wesentliche Vorteile:</span><span class="sxs-lookup"><span data-stu-id="c325e-140">Deploying a Self-contained deployment has two major advantages:</span></span>

- <span data-ttu-id="c325e-143">Sie können sicher sein, dass das Zielsystem Ihre .NET Core-Anwendung ausführen kann, da Sie die .NET Core-Version bereitstellen, die darauf ausgeführt werden soll.</span><span class="sxs-lookup"><span data-stu-id="c325e-143">You can be assured that the target system can run your .NET Core app, since you're providing the version of .NET Core that it will run on.</span></span>

  <span data-ttu-id="c325e-147">Ab .NET Core 2.0 können Sie die Größe Ihrer Bereitstellung auf Linux-Systemen um etwa 28 MB reduzieren, indem Sie den [*invarianten Globalisierungsmodus*](https://github.com/dotnet/corefx/blob/master/Documentation/architecture/globalization-invariant-mode.md) von .NET Core verwenden.</span><span class="sxs-lookup"><span data-stu-id="c325e-147">Starting with .NET Core 2.0, you can reduce the size of your deployment on Linux systems by approximately 28 MB by using .NET Core [*globalization invariant mode*](https://github.com/dotnet/corefx/blob/master/Documentation/architecture/globalization-invariant-mode.md).</span></span> <span data-ttu-id="c325e-148">Normalerweise basiert die Globalisierungsunterstützung von .NET Core unter Linux auf den [ICU-Bibliotheken](https://github.com/dotnet/docs/issues/http%22//icu-project.org).</span><span class="sxs-lookup"><span data-stu-id="c325e-148">Ordinarily, .NET Core on Linux relies on the [ICU libraries](https://github.com/dotnet/docs/issues/http%22//icu-project.org) for globalization support.</span></span> <span data-ttu-id="c325e-149">Im invarianten Modus werden die Bibliotheken nicht in die Bereitstellung einbezogen, und alle Kulturen verhalten sich wie die [invariante Kultur](xref:System.Globalization.CultureInfo.InvariantCulture?displayProperty=nameWithType).</span><span class="sxs-lookup"><span data-stu-id="c325e-149">In invariant mode, the libraries are not included with your deployment, and all cultures behave like the [invariant culture](xref:System.Globalization.CultureInfo.InvariantCulture?displayProperty=nameWithType).</span></span>

- <span data-ttu-id="c325e-150">Das Bereitstellen von zahlreichen eigenständigen .NET Core-Anwendungen auf ein System kann viel Speicherplatz verbrauchen, da jede Anwendung .NET Core-Dateien dupliziert.</span><span class="sxs-lookup"><span data-stu-id="c325e-150">Deploying numerous self-contained .NET Core apps to a system can consume significant amounts of disk space, since each app duplicates .NET Core files.</span></span>
