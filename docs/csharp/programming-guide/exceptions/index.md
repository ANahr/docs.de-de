<span data-ttu-id="18319-103">Die C#-Funktionen zur Ausnahmebehandlung helfen Ihnen dabei, unerwartete oder außergewöhnliche Situationen zu verarbeiten, die beim Ausführen von Programmen auftreten können.</span><span class="sxs-lookup"><span data-stu-id="18319-103">The C# language's exception handling features help you deal with any unexpected or exceptional situations that occur when a program is running.</span></span> <span data-ttu-id="18319-104">Die Ausnahmebehandlung verwendet die Schlüsselwörter `try`, `catch` und `finally`, um Aktionen zu testen, die möglicherweise nicht erfolgreich sind, um Fehler zu behandeln, wenn Sie entscheiden, dass dies vernünftig ist, und um später Ressourcen zu bereinigen.</span><span class="sxs-lookup"><span data-stu-id="18319-104">Exception handling uses the `try`, `catch`, and `finally` keywords to try actions that may not succeed, to handle failures when you decide that it is reasonable to do so, and to clean up resources afterward.</span></span> <span data-ttu-id="18319-105">Ausnahmen können von der Common Language Runtime (CLR), vom .NET Framework bzw. anderen Drittanbieterbibliotheken oder vom Anwendungscode generiert werden.</span><span class="sxs-lookup"><span data-stu-id="18319-105">Exceptions can be generated by the common language runtime (CLR), by the .NET Framework or any third-party libraries, or by application code.</span></span> <span data-ttu-id="18319-106">Ausnahmen werden mit dem Schlüsselwort `throw` erstellt.</span><span class="sxs-lookup"><span data-stu-id="18319-106">Exceptions are created by using the `throw` keyword.</span></span>  
  
 <span data-ttu-id="18319-107">In vielen Fällen wird eine Ausnahme nicht von einer Methode ausgelöst, die von Ihrem Code direkt aufgerufen wurde, sondern von einer anderen Methode weiter unten in der Aufrufliste.</span><span class="sxs-lookup"><span data-stu-id="18319-107">In many cases, an exception may be thrown not by a method that your code has called directly, but by another method further down in the call stack.</span></span> <span data-ttu-id="18319-108">In diesem Fall entlädt die CLR die Liste, sucht nach einer Methode mit einem `catch`-Block für den spezifischen Ausnahmetyp und führt den ersten gefundenen `catch`-Block aus.</span><span class="sxs-lookup"><span data-stu-id="18319-108">When this happens, the CLR will unwind the stack, looking for a method with a `catch` block for the specific exception type, and it will execute the first such `catch` block that if finds.</span></span> <span data-ttu-id="18319-109">Wenn kein geeigneter `catch`-Block in der Aufrufliste gefunden wird, beendet die CLR den Prozess und zeigt eine Meldung für den Benutzer an.</span><span class="sxs-lookup"><span data-stu-id="18319-109">If it finds no appropriate `catch` block anywhere in the call stack, it will terminate the process and display a message to the user.</span></span>  
  
 <span data-ttu-id="18319-110">In diesem Beispiel testet eine Methode, ob eine Division durch Null möglich ist, und fängt den Fehler ab.</span><span class="sxs-lookup"><span data-stu-id="18319-110">In this example, a method tests for division by zero and catches the error.</span></span> <span data-ttu-id="18319-111">Ohne Ausnahmebehandlung würde dieses Programm mit dem Fehler **DivideByZeroException wurde nicht behandelt** beendet.</span><span class="sxs-lookup"><span data-stu-id="18319-111">Without the exception handling, this program would terminate with a **DivideByZeroException was unhandled** error.</span></span>

## <a name="exceptions-overview"></a><span data-ttu-id="18319-113">Übersicht über Ausnahmen</span><span class="sxs-lookup"><span data-stu-id="18319-113">Exceptions Overview</span></span>  
 <span data-ttu-id="18319-114">Ausnahmen weisen folgende Eigenschaften auf:</span><span class="sxs-lookup"><span data-stu-id="18319-114">Exceptions have the following properties:</span></span>  
  
-   <span data-ttu-id="18319-115">Bei Ausnahmen handelt es sich um Typen, die alle letztlich von `System.Exception` abgeleitet werden.</span><span class="sxs-lookup"><span data-stu-id="18319-115">Exceptions are types that all ultimately derive from `System.Exception`.</span></span>  
  
-   <span data-ttu-id="18319-116">Verwenden Sie einen `try`-Block um die Anweisungen, die möglicherweise Ausnahmen auslösen.</span><span class="sxs-lookup"><span data-stu-id="18319-116">Use a `try` block around the statements that might throw exceptions.</span></span>  
  
-   <span data-ttu-id="18319-117">Sobald eine Ausnahme im `try`-Block auftritt, springt der Steuerungsfluss zum ersten zugeordneten Ausnahmehandler, der sich an einer beliebigen Stelle in der Aufrufliste befindet.</span><span class="sxs-lookup"><span data-stu-id="18319-117">Once an exception occurs in the `try` block, the flow of control jumps to the first associated exception handler that is present anywhere in the call stack.</span></span> <span data-ttu-id="18319-118">In C# wird das Schlüsselwort `catch` zum Definieren eines Ausnahmehandlers verwendet.</span><span class="sxs-lookup"><span data-stu-id="18319-118">In C#, the `catch` keyword is used to define an exception handler.</span></span>  
  
-   <span data-ttu-id="18319-119">Wenn für eine bestimmte Ausnahme kein Ausnahmehandler vorhanden ist, beendet das Programm die Ausführung mit einer Fehlermeldung.</span><span class="sxs-lookup"><span data-stu-id="18319-119">If no exception handler for a given exception is present, the program stops executing with an error message.</span></span>  
  
-   <span data-ttu-id="18319-120">Fangen Sie Ausnahmen nur dann ab, wenn Sie sie behandeln und die Anwendung in einem bekannten Zustand belassen können.</span><span class="sxs-lookup"><span data-stu-id="18319-120">Do not catch an exception unless you can handle it and leave the application in a known state.</span></span> <span data-ttu-id="18319-121">Wenn Sie `System.Exception` abfangen, lösen Sie diese mithilfe des Schlüsselworts `throw` am Ende des `catch`-Blocks erneut aus.</span><span class="sxs-lookup"><span data-stu-id="18319-121">If you catch `System.Exception`, rethrow it using the `throw` keyword at the end of the `catch` block.</span></span>  
  
-   <span data-ttu-id="18319-122">Wenn ein `catch`-Block eine Ausnahmevariable definiert, können Sie diese verwenden, um weitere Informationen zum Typ der aufgetretenen Ausnahme abzurufen.</span><span class="sxs-lookup"><span data-stu-id="18319-122">If a `catch` block defines an exception variable, you can use it to obtain more information about the type of exception that occurred.</span></span>  
  
-   <span data-ttu-id="18319-123">Ausnahmen können von einem Programm mithilfe des Schlüsselworts `throw` explizit generiert werden.</span><span class="sxs-lookup"><span data-stu-id="18319-123">Exceptions can be explicitly generated by a program by using the `throw` keyword.</span></span>  
  
-   <span data-ttu-id="18319-124">Ausnahmeobjekte enthalten detaillierte Informationen über den Fehler, z.B. den Zustand der Aufrufliste und eine Textbeschreibung des Fehlers.</span><span class="sxs-lookup"><span data-stu-id="18319-124">Exception objects contain detailed information about the error, such as the state of the call stack and a text description of the error.</span></span>  
  
-   <span data-ttu-id="18319-125">Code in einem `finally`-Block wird auch dann ausgeführt, wenn eine Ausnahme ausgelöst wurde.</span><span class="sxs-lookup"><span data-stu-id="18319-125">Code in a `finally` block is executed even if an exception is thrown.</span></span> <span data-ttu-id="18319-126">Verwenden Sie einen `finally`-Block, um Ressourcen freizugeben, beispielsweise um Streams oder Dateien zu schließen, die im `try`-Block geöffnet wurden.</span><span class="sxs-lookup"><span data-stu-id="18319-126">Use a `finally` block to release resources, for example to close any streams or files that were opened in the `try` block.</span></span>  
  
-   <span data-ttu-id="18319-127">Zusätzlich zum strukturierten Win32-Mechanismus zur Ausnahmebehandlung wurden im .NET Framework verwaltete Ausnahmen implementiert.</span><span class="sxs-lookup"><span data-stu-id="18319-127">Managed exceptions in the .NET Framework are implemented on top of the Win32 structured exception handling mechanism.</span></span> <span data-ttu-id="18319-128">Weitere Informationen finden Sie unter [Structured Exception Handling (C/C++)](/cpp/cpp/structured-exception-handling-c-cpp) (Strukturierte Ausnahmebehandlung [C/C++]) und [A Crash Course on the Depths of Win32 Structured Exception Handling](http://go.microsoft.com/fwlink/?LinkId=119654) (Schnellkurs zu den Details der strukturierten Win32-Ausnahmebehandlung).</span><span class="sxs-lookup"><span data-stu-id="18319-128">For more information, see [Structured Exception Handling (C/C++)](/cpp/cpp/structured-exception-handling-c-cpp) and [A Crash Course on the Depths of Win32 Structured Exception Handling](http://go.microsoft.com/fwlink/?LinkId=119654).</span></span>
