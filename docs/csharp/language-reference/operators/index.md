 <span data-ttu-id="b6392-248">Die arithmetischen Operatoren ([+](../../../csharp/language-reference/operators/addition-operator.md), [-](../../../csharp/language-reference/operators/subtraction-operator.md), [\*](../../../csharp/language-reference/operators/multiplication-operator.md), [/](../../../csharp/language-reference/operators/division-operator.md)) können Ergebnisse erzeugen, die sich außerhalb des zulässigen Wertebereichs für den betreffenden numerischen Typ befinden.</span><span class="sxs-lookup"><span data-stu-id="b6392-248">The arithmetic operators ([+](../../../csharp/language-reference/operators/addition-operator.md), [-](../../../csharp/language-reference/operators/subtraction-operator.md), [\*](../../../csharp/language-reference/operators/multiplication-operator.md), [/](../../../csharp/language-reference/operators/division-operator.md)) can produce results that are outside the range of possible values for the numeric type involved.</span></span> <span data-ttu-id="b6392-249">Einzelheiten zu bestimmten Operatoren finden Sie im entsprechenden Abschnitt, grundsätzlich gilt aber:</span><span class="sxs-lookup"><span data-stu-id="b6392-249">You should refer to the section on a particular operator for details, but in general:</span></span>  
  
- <span data-ttu-id="b6392-250">Arithmetischer Überlauf bei ganzen Zahlen löst entweder eine <xref:System.OverflowException> aus oder verwirft die höchstwertigen Bits des Ergebnisses.</span><span class="sxs-lookup"><span data-stu-id="b6392-250">Integer arithmetic overflow either throws an <xref:System.OverflowException> or discards the most significant bits of the result.</span></span> <span data-ttu-id="b6392-251">Division ganzer Zahlen durch Null löst immer eine <xref:System.DivideByZeroException> aus.</span><span class="sxs-lookup"><span data-stu-id="b6392-251">Integer division by zero always throws a <xref:System.DivideByZeroException>.</span></span>  

   <span data-ttu-id="b6392-252">Wenn ein Überlauf bei ganzen Zahlen auftritt, hängen die Auswirkungen vom Ausführungskontext ab, bei dem es sich um [checked oder unchecked](../../../csharp/language-reference/keywords/checked-and-unchecked.md) handeln kann.</span><span class="sxs-lookup"><span data-stu-id="b6392-252">When integer overflow occurs, what happens depends on the execution context, which can be [checked or unchecked](../../../csharp/language-reference/keywords/checked-and-unchecked.md).</span></span> <span data-ttu-id="b6392-253">In einem "checked"-Kontext wird eine <xref:System.OverflowException> ausgelöst.</span><span class="sxs-lookup"><span data-stu-id="b6392-253">In a checked context, an <xref:System.OverflowException> is thrown.</span></span> <span data-ttu-id="b6392-254">In einem "unchecked"-Kontext werden die höchstwertigen Bits verworfen, und die Ausführung wird fortgesetzt.</span><span class="sxs-lookup"><span data-stu-id="b6392-254">In an unchecked context, the most significant bits of the result are discarded and execution continues.</span></span> <span data-ttu-id="b6392-255">Bei C# haben Sie die Wahl, einen Überlauf zu verarbeiten oder zu ignorieren.</span><span class="sxs-lookup"><span data-stu-id="b6392-255">Thus, C# gives you the choice of handling or ignoring overflow.</span></span> <span data-ttu-id="b6392-256">Standardmäßig erscheinen arithmetische Operationen in einem *unchecked*-Kontext.</span><span class="sxs-lookup"><span data-stu-id="b6392-256">By default, arithmetic operations occur in an *unchecked* context.</span></span> 

   <span data-ttu-id="b6392-257">Neben arithmetischen Operationen können auch Konvertierungen zwischen ganzzahligen Typen (z.B. die Umwandlung von [long](../../../csharp/language-reference/keywords/long.md) nach [int](../../../csharp/language-reference/keywords/int.md)) einen Überlauf verursachen. Sie sind von der Ausführung „checked“ oder „unchecked“ abhängig.</span><span class="sxs-lookup"><span data-stu-id="b6392-257">In addition to the arithmetic operations, integral-type to integral-type casts can cause overflow (such as when you cast a [long](../../../csharp/language-reference/keywords/long.md) to an [int](../../../csharp/language-reference/keywords/int.md)), and are subject to checked or unchecked execution.</span></span> <span data-ttu-id="b6392-258">Bitweise Operatoren und Schiebeoperatoren verursachen allerdings nie einen Überlauf.</span><span class="sxs-lookup"><span data-stu-id="b6392-258">However, bitwise operators and shift operators never cause overflow.</span></span>  
   
-   <span data-ttu-id="b6392-259">Arithmetischer Überlauf oder Division durch 0 (null) löst bei Gleitkommazahlen nie eine Ausnahme aus, weil die Gleitkommatypen auf IEEE 754 basieren und daher Vorkehrungen für die Darstellung von Unendlich und NaN (Not a Number/Keine Zahl) aufweisen.</span><span class="sxs-lookup"><span data-stu-id="b6392-259">Floating-point arithmetic overflow or division by zero never throws an exception, because floating-point types are based on IEEE 754 and so have provisions for representing infinity and NaN (Not a Number).</span></span>
