 <span data-ttu-id="cb2a8-105">Empfohlen: Dieser Fehler kann beim Implementieren einer Schnittstelle auftreten, die Methodennamen enthält, die im Konflikt mit den automatisch generierten Methoden stehen, die mit einer Eigenschaft oder einem Ereignis verknüpft sind.</span><span class="sxs-lookup"><span data-stu-id="cb2a8-105">Suggested: This error can occur when implementing an interface that contains method names which conflict with the auto-generated methods associated with a property or event.</span></span> <span data-ttu-id="cb2a8-106">Die Get/Set-Methoden für Eigenschaften werden als „get_property“ und „set_property“ generiert, und die Methoden zum Hinzufügen und Entfernen von Ereignisse werden als „add_event“ und „remove_event“ generiert.</span><span class="sxs-lookup"><span data-stu-id="cb2a8-106">The get/set methods for properties are generated as get_property and set_property, and the add/remove methods for events are generated as add_event and remove_event.</span></span> <span data-ttu-id="cb2a8-107">Wenn eine Schnittstelle beide Methoden enthält, tritt ein Konflikt auf.</span><span class="sxs-lookup"><span data-stu-id="cb2a8-107">If an interface contains either of these methods, a conflict occurs.</span></span> <span data-ttu-id="cb2a8-108">Um diesen Fehler zu vermeiden, implementieren Sie die Methoden mit einer expliziten Schnittstellenimplementierung.</span><span class="sxs-lookup"><span data-stu-id="cb2a8-108">To avoid this error, implement the methods using an explicit interface implementation.</span></span> <span data-ttu-id="cb2a8-109">Geben Sie dazu die Funktion wie folgt an:</span><span class="sxs-lookup"><span data-stu-id="cb2a8-109">To do this, specify the function as:</span></span>

  
## <a name="example"></a><span data-ttu-id="cb2a8-110">Beispiel</span><span class="sxs-lookup"><span data-stu-id="cb2a8-110">Example</span></span>  
 <span data-ttu-id="cb2a8-111">Im folgenden Beispiel wird der Fehler CS0686 generiert:</span><span class="sxs-lookup"><span data-stu-id="cb2a8-111">The following sample generates CS0686:</span></span>  
  
```csharp  
// CS0686.cs  
interface I  
{  
    int get_P();  
}  
  
class C : I  
{  
    public int P  
    {  
        get { return 1; }  // CS0686  
    }  
}  
// But the following is valid:  
class D : I  
{  
    int I.get_P() { return 1; }  
    public static void Main() {}  
}  
```  
  
## <a name="example"></a><span data-ttu-id="cb2a8-112">Beispiel</span><span class="sxs-lookup"><span data-stu-id="cb2a8-112">Example</span></span>  
 <span data-ttu-id="cb2a8-113">Dieser Fehler kann auch beim Deklarieren von Ereignisse auftreten.</span><span class="sxs-lookup"><span data-stu-id="cb2a8-113">This error can also occur when declaring events.</span></span>  <span data-ttu-id="cb2a8-114">Das Ereigniskonstrukt generiert die Methoden `add_``event` und `remove_``event` automatisch. Das kann zu einem Konflikt mit den gleichnamigen Methoden in einer Schnittstelle führen, wie im folgenden Beispiel dargestellt:</span><span class="sxs-lookup"><span data-stu-id="cb2a8-114">The event construct automatically generates the `add_``event` and `remove_``event` methods, which could conflict with the methods of the same name in an interface, as in the following sample:</span></span>  
  
```csharp  
// CS0686b.cs  
using System;  
  
interface I  
{  
    void add_OnMyEvent(EventHandler e);  
}  
  
class C : I  
{  
    public event EventHandler OnMyEvent  
    {  
        add { }  // CS0686  
        remove { }  
    }  
}  
  
// Correct (using explicit interface implementation):

{  
    void I.add_OnMyEvent(EventHandler e) {}  
    public static void Main() {}  
}  
```
