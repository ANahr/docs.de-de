<span data-ttu-id="14b69-103">Ein bitweiser ODER-Operator wird für einen signaturerweiterten Operanden verwendet. Sie sollten zuerst in einen kleineren unsignierten Typ konvertieren.</span><span class="sxs-lookup"><span data-stu-id="14b69-103">Bitwise-or operator used on a sign-extended operand; consider casting to a smaller unsigned type first</span></span>  
  
 <span data-ttu-id="14b69-104">Der Compiler hat eine Variable implizit erweitert und signaturerweitert und den sich daraus ergebenden Wert in einer bitweisen ODER-Operation verwendet.</span><span class="sxs-lookup"><span data-stu-id="14b69-104">The compiler implicitly widened and sign-extended a variable, and then used the resulting value in a bitwise OR operation.</span></span> <span data-ttu-id="14b69-105">Dies kann zu unerwartetem Verhalten führen.</span><span class="sxs-lookup"><span data-stu-id="14b69-105">This can result in unexpected behavior.</span></span>  
  
 <span data-ttu-id="14b69-106">Im folgenden Beispiel wird CS0675 generiert:</span><span class="sxs-lookup"><span data-stu-id="14b69-106">The following sample generates CS0675:</span></span>  
  
```csharp  
// CS0675.cs  
// compile with: /W:3  
using System;  
  
public class sign  
{  
   public static void Main()  
   {  
      int hi = 1;  
      int lo = 1;  
      long value = (((long)hi) << 32) | lo;              // CS0675  
      // try the following line instead  
      // long value = (((long)hi) << 32) | ((uint)lo);   // correct  
   }  
}  
```
