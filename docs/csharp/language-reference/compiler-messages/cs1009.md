# <a name="compiler-error-cs1009"></a><span data-ttu-id="cb6de-102">Compilerfehler CS1009</span><span class="sxs-lookup"><span data-stu-id="cb6de-102">Compiler Error CS1009</span></span>
<span data-ttu-id="cb6de-103">Nicht erkannte Escapesequenz</span><span class="sxs-lookup"><span data-stu-id="cb6de-103">Unrecognized escape sequence</span></span>  
  
 <span data-ttu-id="cb6de-104">Ein unerwartetes Zeichen folgt einem umgekehrten Schrägstrich (\\) in einem [string](../../../csharp/language-reference/keywords/string.md).</span><span class="sxs-lookup"><span data-stu-id="cb6de-104">An unexpected character follows a backslash (\\) in a [string](../../../csharp/language-reference/keywords/string.md).</span></span> <span data-ttu-id="cb6de-105">Der Compiler erwartet eines der gültigen Escapezeichen.</span><span class="sxs-lookup"><span data-stu-id="cb6de-105">The compiler expects one of the valid escape characters.</span></span> <span data-ttu-id="cb6de-106">Weitere Informationen finden Sie unter [Escapezeichen](../../../standard/base-types/character-escapes-in-regular-expressions.md).</span><span class="sxs-lookup"><span data-stu-id="cb6de-106">For more information, see [Character Escapes](../../../standard/base-types/character-escapes-in-regular-expressions.md).</span></span>  
  
 <span data-ttu-id="cb6de-107">Im folgenden Beispiel wird CS1009 generiert.</span><span class="sxs-lookup"><span data-stu-id="cb6de-107">The following sample generates CS1009.</span></span>  
  
```csharp  
// CS1009-a.cs  
class MyClass  
{  
   static void Main()  
   {  
      // The following line causes CS1009.  
      string a = "\m";     
      // Try the following line instead.  
      // string a = "\t";  
   }  
}  
```  
  
 <span data-ttu-id="cb6de-108">Eine häufige Ursache für diesen Fehler ist die Verwendung des umgekehrten Schrägstrichs in einem Dateinamen, wie im folgenden Beispiel gezeigt.</span><span class="sxs-lookup"><span data-stu-id="cb6de-108">A common cause of this error is using the backslash character in a file name, as the following example shows.</span></span>  
  
```csharp  
string filename = "c:\myFolder\myFile.txt";  
```  
  
 <span data-ttu-id="cb6de-109">Um diesen Fehler zu beheben, verwenden Sie „\\\\“ oder das @-quoted-Zeichenfolgenliteral, wie im folgenden Beispiel gezeigt.</span><span class="sxs-lookup"><span data-stu-id="cb6de-109">To resolve this error, use "\\\\" or the @-quoted string literal, as the following example shows.</span></span>  
  
```csharp  
// CS1009-b.cs  
class MyClass  
{  
   static void Main()  
   {  
      // The following line causes CS1009.  
      string filename = "c:\myFolder\myFile.txt";     
      // Try one of the following lines instead.  
      // string filename = "c:\\myFolder\\myFile.txt";  
      // string filename = @"c:\myFolder\myFile.txt";  
   }  
}  
```
